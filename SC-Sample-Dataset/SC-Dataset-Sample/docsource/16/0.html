FAQ Android

Q: What is the difference between Couchbase Lite and TouchDB?

TouchDB was recently renamed to Couchbase Lite, and here are the differences:

  • All classes/files have been renamed TDObject -> CBLObject
  • Couchbase Lite is geared towards using Android Studio
  • Project structure has been re-organized to fit into the Android Studio way of doing things, especially the tests.
  • The PhoneGap plugin (LiteGap) uses Couchbase Lite as opposed to TouchDB
  • All new development from Couchbase is happening on Couchbase Lite
  • A few bug fixes have happened on Couchbase Lite, with no plans to backport them to TouchDB

Other than that, at this point there's not actually that much difference in the codebases. However the codebases are expected to diverge.

It's currently unclear what will happen to TouchDB in terms of maintenance, at the moment there is no known "corporate sponsor" or active community behind it.

Q: Which versions of Android are supported

Android 2.3 Gingerbread (API level 9) and above.

Q: How can I build Couchbase-Lite projects with Eclipse?

Android Studio is the current recommended IDE, however if you are required to use Eclipse, here is some information that discusses the current situation with Eclipse compatibility.

Since Eclipse + Maven plugin cannot handle .aar dependencies, and the Couchbase-Lite libraries are packaged as .aar's, there is no way to currently use the Maven dependencies.

The two alternatives for including dependencies are:

  • Direct code dependency
  • Direct Jar dependency

NOTE: It may be possible to export projects to Eclipse via the Gradle Eclipse plugin, which can create correct Eclipse projects just by adding apply plugin: 'eclipse' in each project's build.gradle file, and then from the root just run "gradle eclipse" and it will create the project files ready to import.

See Building In Eclipse in the Android Bootstrap README, which might have relevant information.

Q: Where are the Couchbase-Lite Maven artifacts hosted? I don't see them in Maven Central.

They are not in Maven Central yet, but we are planning to upload them into Maven Central at some point.

Q: What are all these different sub-projects for?

A: Couchbase Lite is designed to be modular, allowing the application developer to choose the pieces they need and distribute the smallest application possible.

  • CBLite - This is the core of Couchbase Lite. It is required for all Couchbase Lite applications.
  • CBLiteJavaScript - This is a module which allows for compilation/execution of view map/reduce functions written in the JavaScript language. Native applications typically do not require this module as they will implement map/reduce functions in Java. This module enables CouchApp style applications which depend on JavaScript. Use of this module adds approximately 1MB to your application.
  • CBLiteListener - This module allows you to start an HTTP server port connected to Couchbase Lite. Native applications typically don't need this module unless they also wish to support peer-to-peer replication. The module is also useful for CouchApp style applications in a PhoneGap container.
  • CBLiteEktorp - This is an adapter that allows you to use the Ektorp API to talk to Couchbase Lite. This is an unofficial, unsupported module that has served as a placeholder until an official Native API is put in place.

Q: Which version of Ektorp is used?

A: The CBLiteEktorp uses a forked version of Ektorp based on Ektorp 1.2.2.

Q: I have an app that has several different activities. Should I create a couchDbConnector for every activity, or one should create a single couchDbConnector for the whole app?

A: Create the couchDbConnector once in the application, and have the different activities reference it via:

  couchDbConnector = ((MyApplication)getApplication()). couchDbConnector;
It's pretty much the exact same answer as what is described in this stack overflow post

Q: How can I add fields to my JSON documents without getting the error: UnrecognizedPropertyException: Unrecognized field .., not marked as ignorable

A: The Jackson JSON parser supports an annotation @JsonIgnoreProperties(ignoreUnknown=true) which will allow older coder to parse and ignore unknown fields in the JSON.

Q: Why do I see this message when I build Couchbase-Lite-Android The method ... of type ... must override or implement a supertype method

A: Typically this happens when you're not using a 1.6 compiler. Check your Java Compiler settings in Eclipse and choose a 1.6 compiler.

Q: How can I be notified when a document is updated as a result of replicating against a remote server?

A: The CBLDatabase object implements Observable, so if you add an Observer you will get notifications in the form of a Map changeNotification, which will have {"rev": $revIdOfNewlyAddedRevision, "seq": $seqNumberOfNewlyAddedRevision, "source": $urlOfReplicationIfApplicable}

Q: Why is couchbase lite running in the emulator not able to connect to my sync gateway running on 127.0.0.1?

A: It is running on a different network interface, so to connect to the "host" which launched the emulator, you must use the special ip address: 10.0.2.2.

Q: How can I connect to a Sync Gateway via HTTPS which does not have an SSL cert signed by a root authority?

A: The HttpClient used by Couchbase Lite has some pluggability support for the SSL certificate trust. See this pull request for an example on how to do this.

Q: Why do I see these messages when I run my application [2012-05-15 15:27:32 - Couchbase-Lite-Android] Could not find Couchbase-Lite-Android.apk!

A: Remove the preset "Is Libary" from the projects (Projects->Properties->Android->Library) and run each of them (Run As->Android Application). Eclipse might prompt an error message saying there was not found an "Launcher activity" but it has synced the android application package. Do not forget to set the library as "Is Library" afterwards again or you will get another error message when using the packages for example with the TestApp.

Q: Why do I see this message when I run my application java.lang.NoSuchMethodError: android.database.sqlite.SQLiteDatabase.insertWithOnConflict

A: This happens when you run with an SDK < 2.2. Couchbase-Lite-Android requires SDK 2.2 or newer.

Q: Why do I see a message like java.net.MalformedURLException: Unknown protocol: cblite

A: When using the Ektorp adapter we need to install a URL handler for the cblite protocol. This needs to be done early in your application life-cycle prior to using Couchbase Lite or accessing any other URLs. Currently I recommend you place the following static initialization block in your application's main Activity:

{
    CBLURLStreamHandlerFactory.registerSelfIgnoreError();
}

Q: Why do I see messages like Dx warning: Ignoring InnerClasses attribute for an anonymous inner class (Acme.Serve.Main$1) that doesn't come with an associated EnclosingMethod attribute.

A: This has to do with the packaging of the HTTP Server library we're using in the Listener module. I think we can fix this by repackaging it properly in the future. For now it does not seem to cause any other problems and can be ignored.

Q: I see code defining views at runtime. Are views persisted in the database?

A: Because Couchbase Lite defines views using native code, they must be provided to Couchbase Lite at runtime. However, the results of the map function are stored and re-used. When registering your view with Couchbase Lite they are given a name and version number. Views with the same name and version will reuse data already generated. So, when changing the implementation of your views it is important to bump the version number. See Couchbase Lite Ektorp Views for more information.

Q: I cloned the GrocerySync App from GitHub and got the project references updated. Am I supposed to see a splash screen that says "Starting Couchbase-Mobile"?

A: No. This indicates you are using the wrong source base.
