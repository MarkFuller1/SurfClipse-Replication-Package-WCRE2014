    Hello,

     

    I had the same problem, and I manage to solve it !

    First we need to check if it is actually the same problem...

     

    When you enter the "plug in register" (or some name that is close I am in french so I roughly translated ) view in eclipse, look for the plugin called com.adobe.flashbuilder.codemodel. Is it started (green arrow) ?

     

    If not, it means it could not start. To know the reason, you need to see in Eclipse error log an exception that concerns this plugin. For me it was :

    Caused by: java.lang.ClassCastException: com.ctc.wstx.sr.ValidatingStreamReader incompatible with org.codehaus.stax2.XMLStreamReader2

     

    This exception is caused by the fact that adobe com.adobe.flash.codemodel.osgi use some third party package (com.ctc.wstx) by including it directly in it's package definition.You can find the given package in : %Adobe_install%\eclipse\plugins\com.adobe.flash.codemodel.osgi_%versi on%\classes. As they use it without giving any version number, if any other plugin in your eclipse declare the same third party library adobe might use this one... And in any previous to 4.0.0 version of com.ctc.wstx the ValidatingStreamReader inherite org.codehaus.stax2.XMLStreamReader and not org.codehaus.stax2.XMLStreamReader2.

     

    I have made a workaround by creating a plugin version of the adobe third party lib. I can help you through it if this is indeed your problem.

     

    Regards
