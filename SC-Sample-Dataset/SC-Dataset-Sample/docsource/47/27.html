Project Member #1 tnor...@google.com 
What device are you rendering the screen for?

This sounds a lot like a bug specific to rendering in either ldpi or tvdpi, meaning either the Nexus 7 device definition (tvdpi), or one of the smallest old screens (ldpi).

However, this was fixed in revision 3 of the Android 4.1 (API 16) layout library. Can you please open the SDK manager and make sure you have the very latest up to date version of the Android 4.1 SDK?

Project Member #2 tnor...@google.com 
Oh wait, you said that it *did* work for you in API 16, and was broken in older versions. Yes, that's the above bug. The issue was fixed in an update to 4.1 SDK, but the fix has not been backported to older versions of the layout library -- yet. I believe Xav is planning on doing that along with a few other misc layoutlib fixes that have accumulated.
