Pressing Escape to close MIME message returns 'An error has occurred. See error log for more details'

Technote (troubleshooting)


Problem

When you press the Escape key to close a MIME message, the following error will occur:

    "An error has occurred. See error log for more details".

If you go back into the same document, and perform the same operation again, the error may or may not occur.


Cause

This issue appears to be related to or caused by the embedded browser, because the error will not occur when the embedded browser is disabled. The embedded browser is used when rendering MIME messages in the Standard or Basic 8.0.x client.


To disable the browser:

1. In the Notes Client, go to File --> Preferences, "Basic Client Configuration" section.

2. In the "Additional Options" section, locate the option "Disable embedded browser for MIME mail"

To disable the embedded browser for multiple users you can push out the INI parameter BrowserRenderDisable=1 via a desktop policy.


Diagnosing the problem

Collect an ISA from the user's machine to review the trace and error logs. The following Java errors will appear in the Eclipse logs (located in the .\data\workspace\logs directory):


Problems occurred when invoking code from plug-in: "org.eclipse.ui.workbench".
java.lang.IndexOutOfBoundsException


Resolving the problem

This issue is fixed in 8.0.2 Fix Pack 2 and 8.5.1.

Fix details: SPR# JEIN7GLHYU

Refer to the Upgrade Central site for details on upgrading Notes/Domino.
