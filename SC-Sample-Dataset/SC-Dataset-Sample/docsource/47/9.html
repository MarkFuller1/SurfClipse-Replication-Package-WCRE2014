I have investigated this problem by running CDT/Linuxtools under PDE debugger:
These two exceptions are due there is no local perf on host computer. This is not expected behavior, just a bug imho. 
After I installed local perf I was able to fill in launch configuration main page as shown on attached screenshot: set copy executable from local file to rse connection to beagleboard and pressed apply. Then I removed perf on host again to verify if Perf Options Tab will use perf on my target BeagleBoard to obtain info from it and noticed that perf is not found again. 
I investigated this by debugger and noticed that RemoteProxyManager.java which is expected to select connection type based on project settings always gets null from getSchemeFromNature(project) in code fragment below:

public IRemoteFileProxy getFileProxy(IProject project) throws CoreException {
  if (project == null) {
    return getLocalFileProxy(null);
  }
  String scheme = mapping.getSchemeFromNature(project);
  if (scheme!=null) {
    IRemoteProxyManager manager = getRemoteManager(scheme);
    if (manager != null)
      return manager.getFileProxy(project);
  }
  URI projectURI = project.getLocationURI();
  return getFileProxy(projectURI);
}
this leads that local connection is used always because there is no rse mapping defined at extention point RemoteProxyNatureMapping. This is another bug that prevents remote profiling with perf working. Expected behavior is to use remote perf to obtain information for perf tabs.
