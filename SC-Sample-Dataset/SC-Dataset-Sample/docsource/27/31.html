Using the Java sample code

The Java sample code is available for browsing in the repository, and can also be downloaded in a ZIP file.

Prerequisites

You'll need the following packages installed on your system to run the sample code:

  • JDK version 1.5 or higher
  • The JavaMail API. This is available alone or as part of Java EE.
  • Apache Ant
  • An OAuth Token and Token Secret for a Google Mail account. You can use xoauth.py to generate these for your own account.

Downloading and configuring

Download the sample code package from here, unzip it, and change into the xoauth-java-sample-20100617 directory.

Open the file build.properties in a text editor and modify the following properties:

  • javamail_jar: If you are not using a Java EE jdk and need to specifically include the location of the JavaMail JAR file, put that here. Otherwise it's ok to leave this property blank.
  • email: The email address of a Google Mail account to which you want to authenticate.
  • oauthToken: An OAuth token for the Google Mail account.
  • oauthTokenSecret: The OAuth token secret corresponding to oauthToken.

Save build.properties and close your editor

Running the sample app

Execute ant run to run the sample application with the parameters you specified in build.properties. If everything worked properly, you should see some debug output with the concluding line: 

[java] Successfully authenticated to IMAP.

Exploring the sample code

The high-level sample code is in XoauthAuthenticator.java. The two important methods are initialize and connectToImap. initialize installs a SaslClient provider of the XOAUTH mechanism; it needs to be called once before any calls to connectToImap. connectToImap can be called to open new IMAP connections and authenticate to them with XOAUTH.

Another interesting class is XoauthSaslResponseBuilder. It invokes the OAuth library to generate and sign the OAuth request, then formats the result according to the XOAUTH spec.
