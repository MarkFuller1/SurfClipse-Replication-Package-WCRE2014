JavaScript File API, can't access file's content

up vote 0 down vote favorite
1

I have the following code to read several text files and return an array of objects containing these files's details and contents :

var getFilesContents = function( fileInput ) {

    var fileCount = fileInput.files.length,
        results = [ ];

    // Pushing meta datas
    for( var f = 0; f < fileCount; ++f ) {
        var file = fileInput.files[f];
        results.push({
            name   : file.name,
            mime   : file.type,
            format : file.name.match(/\.[0-9a-z]+$/i)[0].substr(1),
            size   : file.size
        });
    }

    // Pushing contents
    for( var f = 0; f < fileCount; ++f ) {
        var reader = new FileReader();
        reader.onload = function() {
            results[f].content = this.result;
        };
        reader.readAsText( fileInput.files[f] );
    }

    return results;
};

The files details are correct, but for some reason the content is always undefined. Am i accessing the right property (this.result) ? What am i doing wrong ?

Thanks for your help !

@Virus721 - Correct, the FileReader and FileAPI functions are by default asynchronous. Your code is written in a Synchronous fashion so yes, you are returning results before you ever set the .content on its items. The type of results is irrelevant for the most part, especially if your code immediately after this function is trying to make use of the results returned. In time if it was a type that is passed by reference then the .content property would be set, but timeouts would be sinfully bad when you should just either program async or use the sync functions. –  Nick Sharp Aug 14 at 20:31
 
@Virus721 Well, it's just that you're not waiting long enough (since the reads are asynchronous). Since you're probably accessing the return value of this function right after calling it, the values at that point are undefined. If you used callbacks, or setTimeout (for testing), you could wait until the array's objects are up to date –  Ian Aug 14 at 20:31
