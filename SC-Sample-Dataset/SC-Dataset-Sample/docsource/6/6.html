Cannot view the realtime Performance Overview charts after upgrading to vCenter Server 5.0 (2009291)

Symptoms

  • After upgrading to vCenter Server 5.0, you cannot see the realtime Performance Overview charts for all ESX/ESXi hosts.
  • Loading Performance Overview chartsfail
  • In vCenter Server, you see the error:

    Chart could not be loaded

  • In the vCenter Server stats.log file, you see the error:

    Caused by: com.vmware.vim.stats.cil.ChartDrawingException: Error while streaming JFreeChart image.
    at com.vmware.vim.stats.impl.cil.jfreechart.JFreeChartImageLibrary$1.write(Unknown Source)
    ... 23 more
    Caused by: javax.imageio.IIOException: Can't create output stream!
    at javax.imageio.ImageIO.write(ImageIO.java:1560)
    at org.jfree.chart.encoders.SunPNGEncoderAdapter.encode(SunPNGEncoderAdapter.java:127)
    at org.jfree.chart.encoders.EncoderUtil.writeBufferedImage(EncoderUtil.java:168)
    ... 24 more
    Caused by: javax.imageio.IIOException: Can't create cache file!
    at javax.imageio.ImageIO.createImageOutputStream(ImageIO.java:397)
    at javax.imageio.ImageIO.write(ImageIO.java:1558)
    ... 26 more
    Caused by: java.io.IOException: The system cannot find the path specified
    at java.io.WinNTFileSystem.createFileExclusively(Native Method)
    at java.io.File.checkAndCreate(File.java:1704)
    at java.io.File.createTempFile(File.java:1792)
    at javax.imageio.stream.FileCacheImageOutputStream.<init>(FileCacheImageOutputStream.java:71)
    at com.sun.imageio.spi.OutputStreamImageOutputStreamSpi.createOutputStreamInstance(OutputStreamImageOutputStreamSpi.java:50)
    at javax.imageio.ImageIO.createImageOutputStream(ImageIO.java:393)
    ... 27 more
    [ERROR com.vmware.vim.stats.webui.chart.StatsChartServlet] Unable to generate performance chart image. Details: Error while drawing chart report.
    com.vmware.vim.stats.exporter.ReportExportException: Error while drawing chart report.
  • In the vCenter Server 5.x log filevctomcat-stderr.XXXX-XX-XX.log, located at the tomcat folder, you see entries similar to:

    2011-10-01 19:13:36 Commons Daemon procrun stderr initialized
    Oct 1, 2011 7:13:45 PM org.apache.catalina.startup.Embedded initDirs
    SEVERE: Cannot find specified temporary folder at C:\Program Files\VMware\Infrastructure\tomcat\temp
    Oct 1, 2011 7:13:49 PM org.apache.catalina.core.AprLifecycleListener init

Cause

This issue occurs when the temp folder within C:\Program Files\VMware\Infrastructure\tomcat is deleted or missing.

Resolution

To resolve this issue, recreate the temporary folder under C:\Program Files\VMware\Infrastructure\tomcat.
Note: Youneed notrestart the vCenter Server services for the changes to take effect.
