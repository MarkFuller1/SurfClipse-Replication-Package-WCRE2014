Game crashing during loading [v14b] (solved)

by drgoofball » Thu Oct 10, 2013 2:19 am

Hello,
Played towns for about a year (v10 and up), my comp has had no problems with the game so far and I'm somewhat able to understand the coding behind the game. The game crashes during loading/initializing (at the "initializing panels" stage) of both new and old saved maps, the following error log is reported:

[8-Oct-2013 9:31:31 PM][UtilsGL] javax.imageio.IIOException: Can't read input file!
[8-Oct-2013 9:33:09 PM][SmartMenu] Error obtaining name of item/building [qwoodennortharrow]

I'm pretty sure the second error is a direct result of the first, but I can't see why there would be a problem reading the input file (I'm guessing items.png?) at this stage in the loading process. I have updated/reinstalled the game and java (tried both 6 & 7 RTEs) several times and even updated my GPU, ran as an admin from towns.exe and, halved the numbers in the towns.cmd file, all with no change. When I execute the towns.cmd I get the following message:

'java' is not recognized as an internal or external command, operable program or batch file.

I have searched for instances of "woodennortharrow"; so far I have graphics.ini and items.xml (note that the south wooden arrow is given the north image in items.png), but I can't find any reference to "qwoodennortharrow" (action command???). I feel like I am missing the file that defines the rotation actions. I have completely removed my towns folder and downloaded from scratch (steam) ... ??? Any help would be greatly appreciated.
Regards

[SOLVED]
Problem with incompatible mod (thought I removed them already, but I had forgot that mods are located in the .towns folder), after actually removing all mods (for now) the game launched successfully.
