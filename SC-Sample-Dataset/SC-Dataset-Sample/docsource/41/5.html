Problem summary

  • ****************************************************************
* USERS AFFECTED:  All users of IBM WebSphere Application Server
****************************************************************
* PROBLEM DESCRIPTION: The encoding of any document used in    *
*                      a transformation is ignored by the      *
*                      debugger.                               *
****************************************************************
* RECOMMENDATION:                                              *
****************************************************************
Previously, the encoding supplied in the transformation input
documents through the stream was not retrieved or utilized
when processing the input. This leads to errors in some
situations.

Problem conclusion

  • The debugger has been fixed to obtain the encoding of the
document from the XML declaration of the stylesheet or
document when possible, or use a default of UTF-8 otherwise.

The fix for this APAR is currently targeted for inclusion in
fix packs  6.1.0.25 and 6.0.2.35.  Please refer to the
Recommended Updates page for delivery information:
http://www.ibm.com/support/docview.wss?rs=180&uid=swg27004980
