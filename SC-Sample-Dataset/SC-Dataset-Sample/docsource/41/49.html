DataSet.WriteXml() doesn't escape special characters

Data Platform Development forums
 > 
XML, System.Xml, MSXML and XmlLite

  • Remove From My Forums

    Question

  • 0

    Hi,

    I'm using a DataSet in which I occasionally store strings with "special character" (i.e. the &, or some accented vowels, like è, ì,à,...), but DataSet.WriteXml() doesn't escape them.
    I.e.: if the DataSet contains the string   " abc < & > abc", the XML I get contains exactly the same string and not something like " abc &lt; &amp; &gt;"..

    What I'm doing wrong?
    I'm using the generated xml as input for a webservice, transforming it with an XSL stylesheet and deserializing the result in the webservice request object.

    Thanks,
    s.

    Accented vowels are not special characters in terms of XML so they do not need to be escaped. The ampersand '&' is special in XML and needs to be escaped as '&amp;', the less-than characters '<' needs to be escaped as '&lt;' and WriteXml does that.

    Here is a C# code sample writing your example:

    Code Block

     

    DataSet ds = new DataSet("Example");

    DataTable t = new DataTable();

    DataColumn c = new DataColumn("col1", typeof(string));

    t.Columns.Add(c);

    ds.Tables.Add(t);

    DataRow row = t.NewRow();

    row["col1"] = "abc < & > abc";

    t.Rows.Add(row);

    ds.WriteXml(Console.Out);

     

    /* result is

    <Example>
      <Table1>
        <col1>abc &lt; &amp; &gt; abc</col1>
      </Table1>
    </Example>

    */

    The exception probably means that there is an XML declaration declaring UTF-8 as the encoding of the XML document but the characters are not properly encoded as UTF-8. So somewhere in your tool chain there is a problem with the encoding introduced. If you use WriteXml to the MemoryStream then make sure you pass the stream directly to the XmlReader or XPathDocument you pass to the Transform method. Don't try to use code to convert that stream to a string.
