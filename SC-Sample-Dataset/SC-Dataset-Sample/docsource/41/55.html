    specification makes it clear (ok, I guess "clear" isn't quite accurate ) that
    attribute values cannot nest quotes of the same type. The HTML specification
    doesn't specifically mention it, but it's well known that you can't do that in
    any modern container.

  5. Re: Problem using Jakarta-taglibs in WLS8.1sp2

    I had a similar problem in that the following code did not work in WLS 8.1
    SP2 (the JSP tags were treated as template text):


    aram name="..." value="<%=request.getParameter("foo")%>"/>



    I had managed to fix it by introducing a temporary variable in a scriptlet:

    <%
    String foo = request.getParameter("foo");
    %>

    aram name="..." value="<%=foo%>"/>



    However, comparing with your example (and solution), it looks like the JSP
    compiler has a problem with nested double quotes.
    (So I should be able to use single quotes in my example above and avoid the
    temporary variable.)

    -- Craig

    According to the jsp spec the quotes need to be escaped when appearing with
    similar type of quotes

    JSP.2.6 in jsp12 spec explains this :

    ----

    A ' is quoted as \'. This is required within a single quote-delimited
    attribute value.

    A " is quoted as \". This is required within a double quote-delimited
    attribute value.

    ---

    The tag would need to look like :

    "/>

    --

    Its unfortunate that the compiler did not throw an exception in this case in
    81 but this is being worked on for the next release and you should have a
    cool compiler which generates meaningful suggestions for such errors on the
    page.

    --Nagesh
