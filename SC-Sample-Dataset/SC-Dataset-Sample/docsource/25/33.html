Map Translator: errors and symptoms

This page is designed to record examples of common Partner errors in various log files and symptoms related to such errors. To find an error message such as those listed below, look in several places, in order:

  1. Open <root translator>/Partner4.0<or Partner4.4>/logs/partner.log and search for fatal, then error
    • An error here indicates there was a problem with the 4.0 Map Viewer or updating the 4.0 map viewer;
  2. Open <root translator>/4.0/logs/partner.log and, again, search for fatal, then error
    • An error here indicates there was a problem somewhere in the Palm/4.0 translation;
  3. Open <root translator>/Shared/data/RoverTranslator.log and search for error
    • This is where the majority of configuration errors will be displayed (labels, find items, transforms, etc).
  4. Finally, open Shared/data/MapTranslator.log and search for error, then stderr
    • This is the only place where you will know if the 3.7 translator needs more memory.
  5. IF ALL ELSE FAILS and the translator is still crashing, you can keep a terminal log up. =>Here is how to do this<=

  4. 4.0: Data doesn't show up in 4.0 viewer
  5. 4.0: editconfig doesn't open Java from the Hub jnlp

  8. 4.0 Icons don't show up 4.0, but do show up in 3.7

  10. 4.0: Lines don't show up in the 4.0 viewer
  11. 4.0: Map Viewer cannot find main class, will not open ==> Installs using Oracle are likely to encounter this problem

  13. 4.0: map viewer (or other java) fails to open after update
  14. 4.0: Map Viewer runs out of memory when opening
  15. 4.0: Turtle Mapset fails to show Turtle meters
  16. 4.0: Weird lines appear and/or lines rotate when hovering
  17. 4.4: Labels don't show up in Partner Viewer
