Thanks to all of you for reporting your crash errors using Mesquite's 
error reporting system.  We are quite pleased that the system is 
showing us various crashes that probably went unreported in the past. 
We've been using the reports to fix bugs in NucleotideComplement, 
StateNamesEditor, and reading character matrices.  We've noticed a 
series of checksum errors, but we will need to issue an update that 
will gather more information to help us understand that.  If you've 
reported crash errors and would like help urgently, please write 
info at mesquiteproject.org.  Otherwise we will fix the bugs as we solve 
them and will have an update ready in the next month or two.

I'd like to ask for any help you can give about two errors that have 
been reported, PrinterException and StackOverflowError.

PrinterException =========================

First, several of you using Windows have had the following error.

java.awt.print.PrinterException: Invalid name of PrintService.
A command could not be completed because an exception occurred (i.e. 
a crash; class java.awt.print.PrinterException ). WARNING: there is a 
chance this crash could have corrupted your data.  Please check your 
data carefully.  If you save any files, you might best use Save As... 
in case file saving doesn't work properly.

sun.awt.windows.WPrinterJob.pageDialog(Unknown Source)

This error seems to be not Mesquite's fault, but rather a bug in 
Java.  However, we're not sure.  For those of you who have 
experienced this problem, does it continue to happen if you go to 
another program or control panel and change default printers?  Does 
it happen every time you try to print a tree?  Does it happen if you 
try to print something else, e.g. a chart or matrix?  What is your 
default printer?
