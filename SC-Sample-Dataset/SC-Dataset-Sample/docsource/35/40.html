 For me this is a bug of HttpClient cache and I think it occurs only in very specific circumstances because usually when you have network issues or a backend that does not respond, you get a ConnectTimeout when opening the connection or a SocketTimeout when sending the request and waiting for the first part of the response with the status line and response headers. Here it looks like we received a part of the response but wait for the rest.

I'll try to work on a patch but I guess this will not be easy to reproduce...
