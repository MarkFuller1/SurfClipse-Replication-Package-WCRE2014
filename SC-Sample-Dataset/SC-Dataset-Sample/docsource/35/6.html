    ZWImport java.net.socketException: socket closed

    After upgrading to Zen 6.5 desktop workstations no longer import, at least
    not more than one at a time. Originally, I recieved java.lang.OutOfMemory
    error when a workstation runs zwsreg.exe. I found documentation suggesting
    the use of the -Xms128M -Xmx256M java command line switches in order to
    allocate more memory. I edited the zenwsimp.ncf file and applied those
    switches appropriately. Now I recieve the error java.net.socketexception:
    socket closed in addition to the outofmemoryerror, though it now takes a
    little longer for the error to appear.
    I believe that this is the root cause of many other problems that I am
    having, such as:
    1.) force run, run once apps do not "run once" but everytime a user logs in.
    2.) remote control, remote view, remote execute, are not working.
    3.) workstation assign applications do not show up on the workstations to
    which they are assigned.

    In otherwords, nothing is working very well since the upgrade.

    other pertinent info:
    Netware 6.5 sp2
    Zen 6.5 desktop
    java.nlm v1.42.03
    All workstations are windows XP Pro with latest service packs.


    Thank you for any help you can offer
    -pl
