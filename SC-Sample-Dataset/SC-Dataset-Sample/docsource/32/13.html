Marco Dubbeld added a comment - 06/Mar/12 06:43 AM

Although according to rfc2616/rfc2616-sec3.html#sec3.4.1 the charset in the content type is without quotes, there are several http stacks around that send charset with quotes. This is not only the case for jax-ws metro stack.

Mule should deal with this when setting the message encoding.

See attached patch which will fixes this issue.
