        • What is the 'right' way to allow arbitrary input in a ComboBox? (IsEditable=true doesn't fire any special event on text input) The SelectionChanged event fires and is then handled whenever the ComboBox is changed using the DropDown Menu. The TextInput event is being generated but isn't making it to your event handler because ComboBox has a TextBox inside of it which is likely marking the event as Handled. One way around this is to use the PreviewTextInput event. Code Snippet public Window1() { InitializeComponent(); // Other handlers here . . . testComboBox.PreviewTextInput += new TextCompositionEventHandler (testComboBox_PreviewTextInput); } void testComboBox_PreviewTextInput( object sender, TextCompositionEventArgs e) { Console .WriteLine( "Text Input: e.Text({0})" , e.Text); Console .WriteLine( "Text Input:> Test Combo Box Text is: ({0}) " , testComboBox.Text); } As long as you are just looking at the input data as it passes by, the PreviewTextInput event should work for your needs. However, if you need to mark the event as handled, you may notice some undesireable behaviors when editing the TextBox. Also, using the TextInput events (as opposed to Keyboard events) seems like it would work better for you because TextInput events also fire with Speech and Handwriting input. Something else to consider would be to implement a method that your PreviewTextInput handler and your SelectionChanged handler both call. This could get rid of duplicate code and if you ever found that TextInput and SelectionChanged didn't handle every event you needed, all that would be needed was an implementation of the event handler for the new input which directly calls the method.
