#370 Swallowed InterruptedException

Dozer v5.3.3
closed-fixed
dmitry (lv)
Performance Issue (5)
5
2012-11-14
2012-04-09
Brabenetz Harald
No

Problem: A Thread which uses the DozerBeanMapper, cannot be cancelled.
Restore of Thread State is required after Catching InterruptedException:

-----------------------------
try {
ready.await();
} catch (InterruptedException e) {
log.error("Thread interrupted: ", e);
// Restore the interrupted status:
Thread.currentThread().interrupt();
}
-----------------------------

There is a very good description about this "Java theory and practice: Dealing with InterruptedException":
http://www.ibm.com/developerworks/java/library/j-jtp05236/index.html

With friendly regards,
Harald
