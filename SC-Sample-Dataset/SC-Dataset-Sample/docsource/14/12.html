      Sep 7th, 2010, 09:45 PM
      Hi,

      First up thanks to everyone that have worked on and contributed to Spring Batch... its quite an impressive feat of software engineering.

      This particular exception not only crashes the job but takes down the tomcat web application within which it is being executed

      In some cases many hundreds of "banana files" will be processed before the exception is thrown and in other cases it will occur in a relatively short number of files. Processing of each file takes in the order of 2-5 minutes, with some few taking up to 25 minutes to process.

      High Level Functional Description of My Spring Batch Use Case
      A Job is created during a scheduled poll, and this job is meant to gather up all the "banana" files for processing and then present them to multiple threads for execution. Each thread is passed a list of "banana" files for processing which belong to a specific "owner".... so a particular thread might be kept busy for processing of a number of "banana" files (likely in the order of 2-10 files).

      You won't trigger java.lang.InterruptedException from anywhere inside the framework as far as I know. It might come from a thread pool shutting down, e.g. when an application is undeployed. Maybe "takes down the tomcat web application" is a cause rather than a symptom?
