Troubleshooting WAS CE Start Failures

Troubleshooting WAS CE start problems.

Port in use error.
WAS CE requires certain ports to be available in order to start. If one of those ports are not available, the start will fail, with no information to the user. To confirm, you should check your WASCE logs for information. The log is located in your WASCE_INSTALL_DIR\var\log\server.log and will have an exception like this:
java.rmi.server.ExportException: 
Port already in use: 1099;
nested exception is:
java.net.BindException: Address already in use: JVM_Bind
You can either free up that port by closing the program that is using it, or configure WAS CE to use another port for that binding. Run netstat -b from a command window to display a listing of ports currently in use. See WAS CE documentation for more information on customizing your configuration.
Server takes too long to start
We allow the server 60 seconds after starting it to be available for deployment. This is 3 times as long as what we see in testing. But if the users system is under powered, or just getting crushed by other things at that moment, it may take longer. If the server is running you should see a black console window stating that it is available.
