Troubleshooting IBM WebSphere Application Server Community Edition (WAS CE) Start Failures

Troubleshooting WebSphere Application Server Community Edition (WAS CE) start problems.

Port in use error.
WebSphere Application Server CE requires certain ports to be available in order to start. If any of those ports are not available, the start will fail, with no information to the user. To confirm, you should check your WebSphere Application Server CE logs for information. The log is located in your WASCE_INSTALL_DIR\var\log\server.log and will have an exception like this:
java.rmi.server.ExportException: 
Port already in use: 1099;
nested exception is:
java.net.BindException: Address already in use: JVM_Bind
You can either free up that port by closing the program that is using it, or configure WebSphere Application Server CE to use another port for that binding. Run netstat -b from a command window to display a listing of ports currently in use. See the WebSphere Application Server CE documentation for more information on customizing your configuration.
