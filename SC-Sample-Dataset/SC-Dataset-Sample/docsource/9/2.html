Hello,For the deadlock, I had the same problem. (deadlock that occurs only sometimes)I solve it by inserting the migration jar file in the manifest, field 'Boot-Class-Path:'Note that I'm not sure it's a good idea to call RedefineClasses() in the premain() method. If you instrument the classes only once, you should do it in the transformer object, method transform()The RedefineClasses() method is a good place to rewrite classes while your application is running, to add or remove functionnalities (profiling etc...).Hope this helps,Sylvainhttp://sylvainmarechal.chez-alice.fr/

What transformation are you performing?  What is the source of the class file bytes you are transforming?  Are you using a transformation engine -- ASM, etc?Have you tried running with all classes verified?  This will detect bad transformations.Note that class loader requirements on invokes still apply when you do redefine -- for example, when instrumenting classes loaded by the boot class loader the instrumentation support classes which are invoked must also be loaded by the boot class loader.It is fine to call redefine from premain.  If you modify classes used in early start-up (Object, Thread, etc) using redefine saves you from from creating loops in the start-up dependencies.Robert FieldSpecification Lead: JVM TI &#38; java.lang.instrumentSun Microsystems
