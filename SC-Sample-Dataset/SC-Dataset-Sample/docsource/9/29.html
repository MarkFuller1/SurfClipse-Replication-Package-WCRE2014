I think this will be related to a mocking or some other kind of library 
that your project uses. For this reason was offline instrumentation 
implemented in JaCoCo.

Not sure when Sonar will have builtin support for this.

If my above suspicion is correct, for now what you can do is find the 
library (PowerMock?) and remove it from the project.

First of all - this problem doesn't relate to difference in versions.

And finally error message seems clear - "Missing or invalid stackmap frames." This means that some information in your class-files is incorrect. One of the explanations - is a usage of another instrumentation tool  (like PowerMock), which might have bug and produce incorrect class-file. For more info you can do a search in internet of "JaCoCo Missing or invalid stackmap frames".

Hope this can help you to solve your issue.
