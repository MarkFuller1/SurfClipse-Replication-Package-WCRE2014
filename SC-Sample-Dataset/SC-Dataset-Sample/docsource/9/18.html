implemented in JaCoCo.

library (PowerMock?) and remove it from the project.

Re: jacoco exceptions

Could you try following:
  1. Prior to execution of Sonar generate JaCoCo dump ( target/jacoco.exec ) on your own without usage of Sonar, i.e. by using jacoco-maven-plugin or any other automated method described in JaCoCo documentation (Ant, plain agent, but not EclEmma).If you'll see exceptions during this step, then problem doesn't relate to Sonar and will be wondering why EclEmma succeeded.
  2. Execute Sonar analysis with "-Dsonar.dynamicAnalysis=reuseReports" and "-Dsonar.jacoco.reportPath=target/jacoco.exec" to import dump file.
And just in order to clarify: Sonar JaCoCo Plugin does exactly those two steps, except that it tries to automatically attach JaCoCo agent during execution of tests ( i.e. automation of first step ). No less, no more - it cannot affect execution of tests ( i.e. lead to exceptions which you provided in first email) if JaCoCo has no affect on it.

Re: jacoco exceptions

BTW if you use different Java in your automated build and in Eclipse, then behaviour obviously can be different. And that can explain why you succeeded with EclEmma - Eclipse uses EJC (Eclipse Java Compiler).


On Wed, May 8, 2013 at 11:40 PM, Evgeny Mandrikov <[hidden email]> wrote:
Could you try following:
  1. Prior to execution of Sonar generate JaCoCo dump ( target/jacoco.exec ) on your own without usage of Sonar, i.e. by using jacoco-maven-plugin or any other automated method described in JaCoCo documentation (Ant, plain agent, but not EclEmma).If you'll see exceptions during this step, then problem doesn't relate to Sonar and will be wondering why EclEmma succeeded.
  2. Execute Sonar analysis with "-Dsonar.dynamicAnalysis=reuseReports" and "-Dsonar.jacoco.reportPath=target/jacoco.exec" to import dump file.
And just in order to clarify: Sonar JaCoCo Plugin does exactly those two steps, except that it tries to automatically attach JaCoCo agent during execution of tests ( i.e. automation of first step ). No less, no more - it cannot affect execution of tests ( i.e. lead to exceptions which you provided in first email) if JaCoCo has no affect on it.
