Javassist

Last updated Mar 14, 2003.

As I mentioned in the introduction to this series, Javassist is a byte code instrumentation library that allows you to modify byte code by injecting Java code that Javassist converts to byte code for you at runtime. If you've ever used the Java reflection APIs then the Javassist APIs will be very comfortable to you.

The process for using Javassist to load a class is to gain access to a ClassPool and then get a compile-time class (CtClass) from the ClassPool. The javassist.ClassPool class provides a static method named getDefault() that creates a ClassPool by searching the system search path, which includes the platform library, extension libraries, and CLASSPATH:

ClassPool cp = ClassPool.getDefault();

Once you have a ClassPool you can invoke its get() method to retrieve a CtClass (compile-time class) representation of the requested class. The CtClass has very similar methods to the Class class that allow you to find methods, annotations, fields, constructors, and so forth, for a class. Thus the CtClass provides all of the reflection capabilities that you find in the standard Java API. Where Javassist differs from reflection is that it allows you to modify the byte code and change the behavior of the class. You can introduce new methods or fields as well as inject code to be executed before or after a method executes.

The CtClass has a method named getMethods() that returns an array of CtMethod instances. CtMethod, and its superclass CtBehavior, have the following interesting methods:

  • insertBefore(): insert the specified code before the method is executed
  • insertAfter(): insert byte code at the end of the method body
  • insertAt(): inserts byte code at the specified line number in the method
  • addParameter(): adds a new parameter to the method signature
  • addLocalVariable(): adds a new local variable to the method
  • addCatch(): adds a catch block for the specified exception type to the method
  • setBody(): sets the bytecode for the entire body of the method

Javassist provides you with all of the tools to implement whatever strategy you want to employ. When tracing code there are two primary strategies:

  • Modify methods by inserting trace calls at the beginning of the method and at the end
  • Rename the method (e.g. methodName$impl) and replace the existing method with a new implementation that starts the trace, invokes the original method, and then ends the trace

In this section I provide examples of both implementations. Listing 1 shows the simpler implementation, the one that inserts trace calls before the method starts and after the method completes.

Listing 1. JavaAssistTransformer2.java

package com.geekcap.openapm.jvm.agent;

import com.geekcap.openapm.trace.TraceManager;
import java.io.IOException;
import java.lang.instrument.ClassFileTransformer;
import java.lang.instrument.IllegalClassFormatException;
import java.lang.instrument.Instrumentation;
import java.security.ProtectionDomain;
import java.util.ArrayList;
import java.util.HashSet;
import java.util.List;
import java.util.Set;
import javassist.ByteArrayClassPath;
import javassist.CannotCompileException;
import javassist.ClassPool;
import javassist.CtClass;
import javassist.CtMethod;
import javassist.NotFoundException;
import org.apache.log4j.Logger;

/**
 * Class file transformer that uses the Javassist bytecode instrumentation library
 * to inject calls to the OpenAPM Tracer to trace the paths of requests through a
 * Java application
 *
 * @author shaines
 */
public class JavaAssistTransformer2 implements ClassFileTransformer
{
    private static final Logger logger = Logger.getLogger( JavaAssistTransformer.class );

    /**
     * The underlying Java Instrumentation object
     */
    protected Instrumentation instrumentation = null;

    /**
     * A Javassist ClassPool object used for loading, and then instrumenting, classes
     */
    protected ClassPool classPool;

    /**
     * The names of the classes that we've already instrumented so that we don't
     * implement them twice
     */
    protected Set<String> instrumentedClasses = new HashSet<String>();

    /**
     * A list of class prefixes that should not be instrumented, e.g. java.lang
     */
    protected List<String> classesToSkip = new ArrayList<String>();


    /**
     * Creates a new JavaAssistTransformer and sets up the Javassist environment
     *
     * @param instrumentation           The Java Instrumentation object to which we
     *                                  add ourselves as a transformer
     */
    public JavaAssistTransformer2( Instrumentation instrumentation )
    {
        // Save our instrumentation object
        this.instrumentation = instrumentation;

        // Create the default ClassPool, which is built from the CLASSPAHT
        classPool = ClassPool.getDefault();

        // Build the list of class prefixes to skip
        classesToSkip.add( "com.geekcap.openapm" );
        classesToSkip.add( "javax." );
        classesToSkip.add( "java." );
        classesToSkip.add( "sun." );
        classesToSkip.add( "com.sun." );
        classesToSkip.add( "org.jdom" );
        classesToSkip.add( "org.apache." );

        // Add our transformer to the list of transformers
        this.instrumentation.addTransformer( this );

        // Start the TraceManager
        TraceManager.getInstance().startTraceManager();
    }

    /**
     * This method is invoked by the JVM for on all transformers to provide a mechanism for
     * the transformer to modify the bytecode of classes as they are being loaded by the
     * JVM.
     *
     * @param loader                        The ClassLoader that is loading this class
     * @param className                     The name of the class being loaded
     * @param classBeingRedefined           If this is triggered by a redefine or retransform,
     *                                      the class being redefined or retransformed; if this is a class load, null
     * @param protectionDomain              The protection domain of the class being defined or redefined
     * @param classfileBuffer               The input byte buffer in class file format - must not be modified
     *
     * @return                              A well-formed class file buffer (the result of the transform), or null if no transform is performed.
     *
     * @throws IllegalClassFormatException  If the input does not represent a well-formed class file
     */
    @Override
    public byte[] transform( ClassLoader loader,
                             String className,
                             Class classBeingRedefined,
                             ProtectionDomain protectionDomain,
                             byte[] classfileBuffer ) throws IllegalClassFormatException
    {
        // The class name contains slashes instead of periods, so use transform those to
        // dots for readability within the trace
        String dotClassName = className.replace( '/', '.' );

        if( logger.isInfoEnabled() )
        {
            logger.info( "Transforming class: " + className );
        }

        // Only instrument a class once
        if( instrumentedClasses.contains( className ) )
        {
            return null;
        }
        instrumentedClasses.add( className );

        // Skip in the list of class prefixes to skip
        for( String classToSkip : classesToSkip )
        {
            if( dotClassName.startsWith( classToSkip ) )
            {
                return null;
            }
        }


        try
        {
            // Create a Javassist CtClass from the bype code
            classPool.insertClassPath( new ByteArrayClassPath( className, classfileBuffer ) );
            CtClass cc = classPool.get( dotClassName );

            // Only instrument unfrozen classes
            if(  ! cc.isFrozen() )
            {
                // We'll add new methods after we modify the existing methods (so that
                // we don't modify the list of methods while we're working on them(
                List<CtMethod> newMethods = new ArrayList<CtMethod>();

                // Instrument the methods
                CtMethod[] methods = cc.getMethods();
                for( int k = 0; k < methods.length; k ++ )
                {
                    // Only instrument methods in this class, not in the super class because
                    // the super class will be instrumented separately
                    if( methods[k].getLongName().startsWith( dotClassName ) )
                    {
                        // Rename the original method to methodName$impl
                        String methodName = methods[k].getName();
                        String longMethodName = methods[k].getLongName();
                        //methods[k].setName( methodName + "$impl" );

                        if( logger.isDebugEnabled() )
                        {
                            logger.debug( "Instrumenting method: " + longMethodName );
                        }

                        methods[ k ].insertBefore( "{ ( ( com.geekcap.openapm.trace.Trace )com.geekcap.openapm.trace.Tracer.getInstance().get() ).addAction( com.geekcap.openapm.trace.MethodAction.Action.START, \"" + longMethodName + "\" ); }" );
                        methods[ k ].insertAfter( "{ ( ( com.geekcap.openapm.trace.Trace )com.geekcap.openapm.trace.Tracer.getInstance().get() ).addAction( com.geekcap.openapm.trace.MethodAction.Action.END, \"" + longMethodName + "\" ); }" );
                    }
                }

                if( logger.isDebugEnabled() )
                {
                    logger.debug( "Returning instrumented class: " + cc.getName() );
                }

                // Return the instrumented bytecode
                byte[] newClassfileBuffer = cc.toBytecode();
                return newClassfileBuffer;
            }
        }
        catch( IOException ioe )
        {
            logger.error( ioe.getMessage() + " transforming class " + className + "; returning uninstrumented class", ioe );
        }
        catch( NotFoundException nfe )
        {
            logger.error( nfe.getMessage() + " transforming class " + className + "; returning uninstrumented class", nfe );
        }
        catch( CannotCompileException cce )
        {
            logger.error( cce.getMessage() + " transforming class " + className + "; returning uninstrumented class", cce );
        }
        catch( Exception e )
        {
            logger.error( "An error occurred during class transformation: " + e.getMessage(), e );
        }

        // Returning null means that we're going to use the uninstrumented bytecode
        return null;
    }
}

The JavaAssistTransformer2 constructor adds the transformer to the Instrumentation instance and it initializes the ClassPool by invoking ClassPool.getDefault(). The classesToSkip list contains the prefixes of classes that should not be instrumented â€“ this is just a place for you to fine tune your instrumentation. Finally it starts the TraceManager that we created in this series.

The transform() method contains the logic that instruments class methods. If you have a byte array that contains the contents of a class then you can create a CtClass from it as follows:

classPool.insertClassPath( new ByteArrayClassPath( className, classfileBuffer ) );
CtClass cc = classPool.get( dotClassName );

The first statement creates a new ClassPath entry from a byte array and then inserts it into the ClassPool. Once the class is in the ClassPool a CtClass can be created by a simple get() method call.

Before modifying the contents of the class, the transform() first checks to see if the class is â€œfrozenâ€?, which means that "the class has been loaded or written out and thus it cannot be modified any more." If the class is modifiable then the transform() method starts the instrumentation process. It iterates over each method, first checking to see if the name of the method starts with the name of the class — this is done to ensure that we're instrumenting just the class and not its super class (the super class will be loaded separately and we can instrument it then.)

It adds the calls to the Tracer in the following two statements:

methods[ k ].insertBefore( "{ ( ( com.geekcap.openapm.trace.Trace )com.geekcap.openapm.trace.Tracer.getInstance().get() ).addAction( com.geekcap.openapm.trace.MethodAction.Action.START, \"" + longMethodName + "\" ); }" );
methods[ k ].insertAfter( "{ ( ( com.geekcap.openapm.trace.Trace )com.geekcap.openapm.trace.Tracer.getInstance().get() ).addAction( com.geekcap.openapm.trace.MethodAction.Action.END, \"" + longMethodName + "\" ); }" );

The only thing that is complicated in the injected code is the fact that Javassist does not support generics so the Trace class must be explicitly cast to its type. The insertBefore() method injects a call to the START action before the method starts and the insertAfter() method injects a call to the END action after the method ends.

Finally, it converts the CtClass to a byte array containing the byte code and returns it to the caller:

byte[] newClassfileBuffer = cc.toBytecode();
return newClassfileBuffer;

Listing 2 shows an implementation of the other strategy.

Listing 2. JavaAssistTransformer.java

package com.geekcap.openapm.jvm.agent;

import com.geekcap.openapm.trace.TraceManager;
import java.io.IOException;
import java.lang.instrument.ClassFileTransformer;
import java.lang.instrument.IllegalClassFormatException;
import java.lang.instrument.Instrumentation;
import java.security.ProtectionDomain;
import java.util.ArrayList;
import java.util.HashSet;
import java.util.List;
import java.util.Set;
import javassist.ByteArrayClassPath;
import javassist.CannotCompileException;
import javassist.ClassPool;
import javassist.CtClass;
import javassist.CtMethod;
import javassist.CtNewMethod;
import javassist.NotFoundException;
import org.apache.log4j.Logger;

/**
 * Class file transformer that uses the Javassist bytecode instrumentation library
 * to inject calls to the OpenAPM Tracer to trace the paths of requests through a
 * Java application
 *
 * @author shaines
 */
public class JavaAssistTransformer implements ClassFileTransformer
{
    private static final Logger logger = Logger.getLogger( JavaAssistTransformer.class );
    
    /**
     * The underlying Java Instrumentation object
     */
    protected Instrumentation instrumentation = null;

    /**
     * A Javassist ClassPool object used for loading, and then instrumenting, classes
     */
    protected ClassPool classPool;

    /**
     * The names of the classes that we've already instrumented so that we don't
     * implement them twice
     */
    protected Set<String> instrumentedClasses = new HashSet<String>();

    /**
     * A list of class prefixes that should not be instrumented, e.g. java.lang
     */
    protected List<String> classesToSkip = new ArrayList<String>();


    /**
     * Creates a new JavaAssistTransformer and sets up the Javassist environment
     *
     * @param instrumentation           The Java Instrumentation object to which we
     *                                  add ourselves as a transformer
     */
    public JavaAssistTransformer( Instrumentation instrumentation )
    {
        // Save our instrumentation object
        this.instrumentation = instrumentation;

        // Create the default ClassPool, which is built from the CLASSPAHT
        classPool = ClassPool.getDefault();

        // Build the list of class prefixes to skip
        classesToSkip.add( "com.geekcap.openapm" );
        classesToSkip.add( "javax." );
        classesToSkip.add( "java." );
        classesToSkip.add( "sun." );
        classesToSkip.add( "com.sun." );
        classesToSkip.add( "org.jdom" );
        classesToSkip.add( "org.apache." );

        // Add our transformer to the list of transformers
        this.instrumentation.addTransformer( this );

        // Start the TraceManager
        TraceManager.getInstance().startTraceManager();
    }

    /**
     * This method is invoked by the JVM for on all transformers to provide a mechanism for
     * the transformer to modify the bytecode of classes as they are being loaded by the
     * JVM.
     *
     * @param loader                        The ClassLoader that is loading this class
     * @param className                     The name of the class being loaded
     * @param classBeingRedefined           If this is triggered by a redefine or retransform,
     *                                      the class being redefined or retransformed; if this is a class load, null
     * @param protectionDomain              The protection domain of the class being defined or redefined
     * @param classfileBuffer               The input byte buffer in class file format - must not be modified
     *
     * @return                              A well-formed class file buffer (the result of the transform), or null if no transform is performed.
     *
     * @throws IllegalClassFormatException  If the input does not represent a well-formed class file
     */
    @Override
    public byte[] transform( ClassLoader loader, 
                             String className,
                             Class classBeingRedefined,
                             ProtectionDomain protectionDomain,
                             byte[] classfileBuffer ) throws IllegalClassFormatException
    {
        // The class name contains slashes instead of periods, so use transform those to
        // dots for readability within the trace
        String dotClassName = className.replace( '/', '.' );

        if( logger.isInfoEnabled() )
        {
            logger.info( "Transforming class: " + className );
        }

        // Only instrument a class once
        if( instrumentedClasses.contains( className ) )
        {
            return null;
        }
        instrumentedClasses.add( className );

        // Skip in the list of class prefixes to skip
        for( String classToSkip : classesToSkip )
        {
            if( dotClassName.startsWith( classToSkip ) )
            {
                return null;
            }
        }


        try
        {
            // Create a Javassist CtClass from the bype code
            classPool.insertClassPath( new ByteArrayClassPath( className, classfileBuffer ) );
            CtClass cc = classPool.get( dotClassName );

            // Only instrument unfrozen classes
            if(  ! cc.isFrozen() )
            {
                // We'll add new methods after we modify the existing methods (so that
                // we don't modify the list of methods while we're working on them(
                List<CtMethod> newMethods = new ArrayList<CtMethod>();

                // Instrument the methods
                CtMethod[] methods = cc.getMethods();
                for( int k = 0; k < methods.length; k ++ )
                {
                    // Only instrument methods in this class, not in the super class because
                    // the super class will be instrumented separately
                    if( methods[k].getLongName().startsWith( dotClassName ) )
                    {
                        // Rename the original method to methodName$impl
                        String methodName = methods[k].getName();
                        String longMethodName = methods[k].getLongName();
                        methods[k].setName( methodName + "$impl" );

                        if( logger.isDebugEnabled() )
                        {
                            logger.debug( "Instrumenting method: " + longMethodName );
                        }

                        // Make a copy of the method with the original name, this will be the
                        // method that we'll modify the body to add calls to the tracer
                        CtMethod newMethod = CtNewMethod.copy( methods[k], methodName, cc, null );

                        //
                        // Create the body of the new method
                        //
                        String returnType = methods[k].getReturnType().getName();
                        StringBuilder methodBody = new StringBuilder( "{\n" );

                        // Add a START action to the Trace
                        methodBody.append( "\t( ( com.geekcap.openapm.trace.Trace )com.geekcap.openapm.trace.Tracer.getInstance().get() ).addAction( com.geekcap.openapm.trace.MethodAction.Action.START, \"" ).append( longMethodName ).append( "\" );\n" );

                        // Invoke the original method
                        methodBody.append( "\t" );
                        if( !"void".equals( returnType ) )
                        {
                            methodBody.append( returnType + " result = " );
                        }
                        methodBody.append( methodName + "$impl($$);\n" );

                        // Add an END action to the TRACE
                        methodBody.append( "\t( ( com.geekcap.openapm.trace.Trace )com.geekcap.openapm.trace.Tracer.getInstance().get() ).addAction( com.geekcap.openapm.trace.MethodAction.Action.END, \"" ).append( longMethodName ).append( "\" );\n" );

                        // Return the result to the caller
                        if( !"void".equals( returnType ) )
                        {
                            methodBody.append( "\treturn result;" );
                        }

                        // Close the method body
                        methodBody.append( "\n}" );

                        // Debug:
                        if( logger.isTraceEnabled() )
                        {
                            logger.trace(  longMethodName + " method body: " + methodBody );
                        }

                        // Set the method body that we built as the body of the new method
                        newMethod.setBody( methodBody.toString() );

                        // Add the new method to our list of new methods
                        newMethods.add( newMethod );
                    }
               }

                // Add the newly constructed methods to the class
                for( CtMethod newMethod : newMethods )
                {
                    cc.addMethod( newMethod );
                }

                if( logger.isDebugEnabled() )
                {
                    logger.debug( "Returning instrumented class: " + cc.getName() );
                }

                // Return the instrumented bytecode
                byte[] newClassfileBuffer = cc.toBytecode();
                return newClassfileBuffer;
            }
        }
        catch( IOException ioe )
        {
            logger.error( ioe.getMessage() + " transforming class " + className + "; returning uninstrumented class", ioe );
        }
        catch( NotFoundException nfe )
        {
            logger.error( nfe.getMessage() + " transforming class " + className + "; returning uninstrumented class", nfe );
        }
        catch( CannotCompileException cce )
        {
            logger.error( cce.getMessage() + " transforming class " + className + "; returning uninstrumented class", cce );
        }
        catch( Exception e )
        {
            logger.error( "An error occurred during class transformation: " + e.getMessage(), e );
        }

        // Returning null means that we're going to use the uninstrumented bytecode
        return null;
    }
}

Listing 2 is almost identical to listing 1 with the exception of the instrumentation code. Listing 2 renames existing methods from methodName to methodName$impl and then creates a new method that executes a START action, invokes the original method, and then executes an END action. The new method is created by copying the original method and then setting the method body:

CtMethod newMethod = CtNewMethod.copy( methods[k], methodName, cc, null );

The challenge in this implementation is handling different return types and dealing with exception handling (which this implementation completely ignores.)
