Unit Test Adapter threw exception: Type is not resolved for member x. But the application under test compiles and works fine.

    This is a repost of a post I did in the Visual C# IDE forum. (the original  Warnings preventing test run after adding new test class)

    I recently installed a copy of Visual Studio Team System over  Visual Studio 2008.

    I am adding a test class  to an old test project. Whenever I run tests in this class I get a test run error with the following warnings.

    Code coverage instrumentation warning while processing file Hunt.EPIC.Utilities.dll:
    Warning VSP2013 : Instrumenting this image requires it to run as a 32-bit process.  The CLR header flags have been updated to reflect this.
    Warning: Test Run deployment issue: The assembly or module 'SourceRouteHeaderExtension' directly or indirectly referenced by the test container 'd:\workspaces\command center\main\tests\hunt.epic.readingsprocessing.unittest\bin\debug\hunt.epic.readingsprocessing.unittest.dll' was not found.


    I cannot find anything on the net about SourceRouteHeaderExtension. I am also perplexed that all my other tests run just fine but this new one doesn't. Maybe I have to change the target of these tests or something?

    Acutally if you click on the test run little yield sign you get these warnings.

    But the actual exception itself in the test run itself throws an exception as such
    "Unit Test Adapter threw exception: Type is not resolved for member 'Hunt.EPIC.Data.ScrewedUpDuplicateKeyException,Hunt.EPIC.Data, Version=4.0.0.46, Culture=neutral, PublicKeyToken=null'.."

    But the project compiles fine and this particular exception declaration has been in there for a long time.

    I have a hunch the warnings are not the problem but the exception that appears in the test run is the actual problem. Why cant it resolve the type? Where does it look to resolve type. It sounds a simple as a misplaced reference or an old DLL, but I can't seem to find it.
