            Justin Bertram             added a comment - 03/May/12 6:12 PM

            Jason, it's not quite that straight-forward.

            The problem only occurs when a client inside the application server uses the RemoteConnectionFactory before a remote client is able to do so. When this happens the connection factory is initialized in such a way that then causes problems for remote clients. If this doesn't happen then there are no problems when remote clients lookup and use RemoteConnectionFactory.

            Furthermore, a client inside the container shouldn't use RemoteConnectionFactory in the first place. It should either use an MDB (if it is consuming messages), JmsXA (if it is sending messages), or ConnectionFactory (if it is consuming messages and doesn't want to use an MDB).

            In any event, once I resolve HORNETQ-884 this will no longer be a problem.
