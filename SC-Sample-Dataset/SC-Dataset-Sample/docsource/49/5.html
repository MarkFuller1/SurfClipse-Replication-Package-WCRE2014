Jens Bleuel added a comment - 03/Aug/11 12:22 PM
In the Input/Update step, there is a thread started in parallel to grab the field names and put them back to the UI.
I assume this happens when the step dialog is closed before the thread is able to grab the data or another timing aspect comes to play.
From the log, it happeens within
Runnable fieldLoader = new Runnable() {
at line: if (!Const.isEmpty(wTable.getText())) {

When fixed, we should investigate other steps about this and try to find a way to avoid this in future steps or external plugins that use this ongoing.
