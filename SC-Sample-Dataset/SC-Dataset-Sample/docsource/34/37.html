    Yann Simon August 7th, 2012 @ 09:58 AM

    The method recover can be used to catch the exception, but it is not practical but it has to return a Response:

    WS.url("http://localhost/doesNotExist")
.get()
.recover{case e: Throwable => new Response(null) {
            override def status = 500
            override lazy val body = e.toString
          }
        }
.map{case r if r.status = 500 => handle the error}

    It would be easier to have an API like:

    WS.url("http://localhost/doesNotExist")
.get()
.map{Some("ok")}
.onFailure{None}

    map and onFailure would return the same type (here Some[String])

    Actually, you can defer handling of the error to after the map, for example:

    WS.url("http://localhost/doesNotExist")
  .get()
  .map(_ => Some("ok"))
  .recover{case _ => None}
