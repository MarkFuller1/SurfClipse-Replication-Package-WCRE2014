When using com.oreilly.servlet.MultipartRequest to upload files, I get a java.io.UnsupportedEncodingException: ISO-8859-1 -- What's up?

Created Jun 3, 2001

  • Tweet
tansel halic

The problem is not come out as improper META tag of html. becasuse java is providing the ISO-8859-1 encoding standard. the problem source is com.oreilly package. i look inside the source and i realize that the econding type is written wrong. it is written as "ISO-8859-1". but it must be "ISO8859_1". i corrected it. i could do it because the source code is available.

The same problem appears in Java Mail API. but i cant see the source and cant modify.
