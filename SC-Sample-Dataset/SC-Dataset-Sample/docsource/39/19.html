    Hi qforce,

    The encoding "iso-8559-1" specified in the meta-tag of this document does not exist. It is meant to be "iso-8859-1".

    Version 2.5 of the parser has substantially improved the way it handles encoding issues, and recognises this type of problem. It will parse the document using ISO-8859-1 encoding instead of throwing the exception.

    You can download a development release of 2.5 here:
    http://jerichohtml.sourceforge.net/temp/jericho-html-2.5-dev.zip

    Cheers
    Martin
