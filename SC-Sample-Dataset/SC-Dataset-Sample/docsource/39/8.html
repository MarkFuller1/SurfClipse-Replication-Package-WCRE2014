After installng 1.4.1 and the Developer tools I find that using the command
-Dfile.encoding="Latin-1" gives me the following error:

Error occurred during initialization of VM
java.lang.Error: java.io.UnsupportedEncodingException: Latin-1

If I change Latin-1 to ISO-8859-1 there is no problem (1.4.1).

If I run from 1.3.1 with Latin-1 there is no problem.

System OS is 10.2.4

It seems I have badly installed 1.4.1 and/or the Developer tools so I tried
reinstalling 1.4.1 using the download instead of Software Update but
continue having the same problem.

Anyone know why Latin-1 in 1.3.1 works but for 1.4.1 it doesn't and where I
should look to fix it?

FYI the Latin-1 parameter is being passed in Tomcat 4.x.x startup scripts
and I could simply replace all instances of Latin-1 with ISO-8859-1. But I
would like to know why this has happened so I can fix anything else that
might be wrong with the 1.4.1 installation.
