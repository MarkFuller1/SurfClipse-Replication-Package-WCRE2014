    Best Answer - Chosen by Asker

    Well, to me, it sounds like you are trying to bring some value into your program as a numeric datatype (double) and the program is expecting a string, so it is bombing out. It is important to note that any value - and I mean ANY value - brought in from a textbox, dialog, or almost any kind of input method, comes into the program as a string, even if it is a number. Right now, if I ran a program, and typed 24 into an input box, the program would see it as "24" - a string representing 24. To use it as a number, I would need to convert, or cast it, to a double, something like

    Double.ParseDouble("24")

    Try using "nextString" instead of "nextDouble" (or something like that) and see if that works. Then - if you need to - parse it to a double. Hope this helps - good luck : )

      Hello.

      This exception is thrown by the "nextDouble" Scanner's method if the next token (from your stream) does not match the Float regular expression, or is out of range.

      The Float regular expression of Java is a little complex if you are not familiar with Grammar notation. Heres the link if you want to check it:
      http://java.sun.com/j2se/1.5.0/docs/api/…

      Any way, the problem is that you are reading something that is NOT a valid double expression.

      Good Luck.
