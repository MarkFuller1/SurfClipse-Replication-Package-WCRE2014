I bet I might know what the problem is.

Are you using JCreator's general output window when running this program? If so, there are issues when using Scanner with it. Disable capture output and it should work.

InputMismatchException is thrown when something other than a valid integer is read by nextInt(). Because of the issues, it's reading in text instead of your numbers.

If that's not the issue, I don't see anything wrong with your code.

Edit: http://www.jcreator.com/forums/index.php?showtopic=1125 The topic where I reported the bug.

CODE

Otherwise, average will always be 0.0 (dividing ints will round down to the next int)
