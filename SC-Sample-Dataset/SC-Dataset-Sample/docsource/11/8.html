    Best Answer - Chosen by Asker

    The problem is in the nextInt() method.
    It is reading an int and only an int from the line.
    But if you take a closer look at the line you will see that the int has a new line character after it.
    the nextInt() method is not reading this. This is therefore causeing a read mismatch after the first int has been processed.

    Change your code as follows
    public void readCar(Scanner scan) {
    carName = scan.nextLine();
    carNum = scan.nextLine();
    carValue = scan.nextInt(); //Error: Car.readCar(Car.java:15)
    if (scan.hasNext())
    dummy = scan.nextLine();
    }

    You need to declare the String dummy too.

    Have fun.

      It seems you have met this issue:
      http://www.daniweb.com/software-developm…

      When the Scanner gives you a nextInt(), it leaves a newline character in the input, so the next nextLine() returns an empty string and discards the newline. nextInt() skips over any leading whitespace but nextLine does not, causing misalignment:
      ["SUBARU WRX", "SLB007GP", 10500], ["", "HONDA Type-R", -input mismatch-]
