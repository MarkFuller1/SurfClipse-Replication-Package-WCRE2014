Best Answer - Chosen by Voters

The problem is caused in this bit:
while (number != 0)
{
number = scan.nextInt();
numList[index] = number;
index++;
}

specifically where write: number = scan.nextInt(); since the user might not actually type in a number, they may time in some string of text.

A solution to this may be to surround it with a try, catch block.

while (number != 0)
{
try {
number = scan.nextInt();
numList[index] = number;
index++;
}
catch (InputMismatchException err) {
System.out.println("Invalid input! Please enter a number: ");
scan.nextLine();
}
}

You will have to import java.util.InputMismatchException at the beginning.
scan.nextLine(); is important in the catch block, otherwise it won't move beyond the non-digits.
