    java.lang.OutOfMemoryError: Java heap space
    Iman_Paryudi asked Aug 22, 2012 | Replies (10)
    I am creating a program that will do 67,108,864 calculations using Netbeans. But when I run it, there is an error message: java.lang.OutOfMemoryError: Java heap space.

    I have tried several things to overcome this problem:
    1. I changed the setting of VM Options of the Project Properties. I even set the VM Options to -Xmx1024m.
    2. I changed the setting of VM Options of the Tomcat Server. Here, I also set the VM Options to -Xmx1024m.
    3. I disabled the HTTP Monitor.

    What I have not tried is changing the content of netbeans.conf file since my computer does not allow me to change it.

    So, my questions are:

    1. How can I fix this kind of problem in a servlet project?
    2. What is the maximum value of Xmx? Is it the capacity of the RAM?
    3. How can I change the content of netbeans.conf file?

    Environment:

    8 GB RAM

    dkiwi replied Aug 22, 2012
    Hi,

    Java OOM usually results of two main reasons. (1) JVM heap size too low or (2) memory leakage in the code. I am not sure but seems you are case (2) 'cause you have tried to set Xmx.

    To tackle case (2) issue probably you need to optimize our code make sure objects are eligible for GC (garbage collection) so that the program won't use up the memory. In worse scenario you may need to redesign your approach to control the memory usage in the program.

    Shashank replied Aug 29, 2012
    First of all, you are creating way too many PreparedStatement objects, which you are not cleaning up, i.e. you should call prepStateTrainingData.close() at the end of the innermost loop.

    An even more efficient way to go about will be to create the PreparedStatement object outside the loop, and the assign the values within the loop, then call prepStateTrainingData.addBatch(), and after the loop, call prepStateTrainingData.executeBatch(). If you think that the loops are going to create millions of rows, then you can add a counter and call addBatch() after say every 1000 rows to clear out the buffer.

    Either way, never forget to call close() on the PreparedStatement after you're done using it, so that the GC can release the memory and the exception will be resolved!
