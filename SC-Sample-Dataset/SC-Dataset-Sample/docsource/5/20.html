The first question to me is "Should your application need a larger size at that point?" if the answer is no, then is that library stuck in some infinite loop or something else causing it to keep building memory. If yes, then open project.properties and add

run.args.extra=-J-Xmx128m

or what ever you need. See:

http://wiki.netbeans.org/NetBeansDeveloperFAQ
http://wiki.netbeans.org/HowToPassCommandLineArgumentsToANetBeansPlatformApplicationWhenRunInsideTheIDE

Hello.

Quote:
If yes, then open project.properties and add
run.args.extra=-J-Xmx128m

This works only after Clean&Build suite.
After changes in some sub-module and run/debug, this option is ignored.

Add
default_options="-J-Xms24m -J-Xmx1024m"
in \NetBeans\harness\etc\app.conf

But this ignored some time on Linux,
To resolve it - change property "run.args.common" in section <target name="-init"> in
\NetBeans\harness\run.xml
for example:
<property name="run.args.common" value='--userdir "${test.user.dir}" -J-Dnetbeans.logger.console=true -J-ea -J-Xmx1024m -J-XX:MaxPermSize=512m'/>
