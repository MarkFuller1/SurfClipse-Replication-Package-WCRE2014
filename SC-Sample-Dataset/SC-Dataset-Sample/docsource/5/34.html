    OutOfMemoryError: Java heap space while drill down on dimension

    I am using Mondrian 3.3.0, I am getting java.lang.OutOfMemoryError: Java heap space while drilling down a dimension.

    My problem is not out of memory as I know because of the diverse dataset I am using this is very much possible, However I want to prevent OutOfMemoryError and show user mondrian.olap.MemoryLimitExceededException. Though I have configured thresold and enabled memory monitor in configuration, It gives me OutOfMemoryError.

    Any help? What does Mondrian do in case of OutOfMemory?

    My Mondrian configuration is as below
    ==========================
    mondrian.rolap.nonempty=true
    mondrian.util.memoryMonitor.enable=true
    mondrian.result.limit=1000000
    mondrian.rolap.queryTimeout=360
    mondrian.rolap.ignoreInvalidMembersDuringQuery=true
    mondrian.util.memoryMonitor.percentage.threshold=20

    I'm not Java guru, but I was getting heap errors parsing a 545MB XML file. Solution was to edit Spoon.bat:

    if "%PENTAHO_DI_JAVA_OPTIONS%"=="" set PENTAHO_DI_JAVA_OPTIONS="-Xmx4g" "-XX:MaxPermSize=512m"
