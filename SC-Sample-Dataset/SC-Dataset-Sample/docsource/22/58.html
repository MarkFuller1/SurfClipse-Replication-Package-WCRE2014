Virtual Member Manager (VMM) throws unparseable date format
exception for dates from CUSTOM Ldaps. The dates are supported
and covered under LDAPv3 protocol. But VMM is not supporting
them.

Problem summary
 
  
     
   
  • ****************************************************************
* USERS AFFECTED:  All users of IBM WebSphere Application      *
*                  Server                                      *
****************************************************************
* PROBLEM DESCRIPTION: VMM throws Unparseable date Exception   *
*                      when encountered against date formats   *
*                      supported by custom LDAPs and by        *
*                      LDAPv3.                                 *
****************************************************************
* RECOMMENDATION:  Example timeStamp formats :                 *
*                  yyyyMMddHHmmss[.fraction]Z (Fraction of the *
*                  second is optional.)                        *
*                  yyyyMMddHHmmss.SZ                           *
*                  yyyyMMddHHmmssZ                             *
****************************************************************
VMM should support custom LDAP date formats for CUSTOM LDAP
servers.
Given custom property as well LDAP dateFormat should be
specified.
One will have to run following command in order to support
specified dateFormat by VMM.
$AdminTask setIdMgrCustomProperty {-id <LDAP_REPO_ID> -name
ldapTimestampFormat  -value "yyyyMMddHHmmssZ" }
yyyyMMddHHmmssZ >> may vary as per LDAP's dateFormat!

     
    • 
  
 
  
Problem conclusion
 
  
     
   
  • Note : This timestamp would be applicable for all dates in LDAP,
VMM Will apply and try to parse all dates stored in LDAP with
specified dateFormat.

The fix for this APAR is currently targeted for inclusion in
fix pack 6.1.0.45 , 7.0.0.25 , 8.0.0.5 and 8.5.0.1.  Please
refer to the Recommended Updates
page for delivery information:
http://www.ibm.com/support/docview.wss?rs=180&uid=swg27004980
