        • Alexander Pokahr
          2006-11-02

          Hi Umut,

          there is a build.xml in the distribution, which includes a
          java task to execute the jibx compiler (part of the
          "compile" target).
          As a start, you could just run the build file.

          For better eclipse integration, I use a custom build file,
          with the following target, which I have added as a custom
          builder in eclipse:

          <target name="jibx_compile" description="apply jibx compiler">
              <java classname="org.jibx.binding.Compile" failonerror="true" fork="true">
                  <classpath refid="compile.classpath"/>
                  <classpath location="${build.dir}"/>
                  <!--<arg value="-v" />-->
                  <arg value="${build.dir}/jadex/model/jibximpl/binding.xml" />
              </java>
          </target>

          where ${build.dir} is my output folder, and

          <path id="compile.classpath">
              <fileset dir="lib" includes="*.jar" />
          </path>

          Let me know about your progress.

          Also, we would be curious to know some more
          details about your project and plans, if you are
          willing to tell something about it.

          Kind regards,
          Alex

                • Alexander Pokahr
                  2006-11-11

                  Hi Umut,

                  I will send you the sources on monday, as I don't
                  have them at hand at the moment.

                  You are right about the agenda actions.
                  As we see it, the agent behaviour appears more
                  consistent, when side-effects (e.g. caused by belief
                  changes) are computed directly, that is before other
                  unrelated actions, such as different plan steps or
                  processing of received messages.

                  But there might be other ways. We didn't experiment
                  a lot with different strategies. The main reason we
                  implemented it this way (besides finding it natural) was,
                  that it quite nicely resembles the execution style we had,
                  before we invented the agenda based interpreter. ;-)

                  Cheers,
                  Alex
