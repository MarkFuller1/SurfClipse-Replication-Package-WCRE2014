Reflection and Integer.valueOf() Method

I was going thru the Reflection related articles and found the following code:

        Field value = Integer.class.getDeclaredField("value");
        value.setAccessible(true);
       
        value.set(1, 2);
        System.out.println(Integer.valueOf(1));
       
        System.out.printf("5-4 => %d%n",5-4);
       
        value.set(1001, 1002);
        System.out.println(Integer.valueOf(1001));

The output was:

2
5-4 => 2
1001

 I was surprised to see the results. I was expecting '1' but i got '2' in the first 2 cases. At least the 3rd result should have been 1002. I was totally wrong.

What's happening here?

  • Java autoboxing caches Integers -128 to 127 (It is because of this caching behaviour few people recommend to use Integer.valueOf() instead of new Integer()).
  • The code used Reflection to change the value of the integer. As a result, the method Integer.valueOf(int i) returned the cached value (ie. modified value) and not the original value. This caused the above Java code's output to appear weired.

On a similar note the following code produced interesting output:

public class StringDestroyer {
public static void main(String[] args)
throws IllegalAccessException, NoSuchFieldException {
Field value = String.class.getDeclaredField("value");
value.setAccessible(true);
value.set("hello!", "cheers".toCharArray());
System.out.println("hello!");
}
}

Output:

cheers

DB2 - SQL1005N The database alias "test" already exists ...

 I tried to use a database named 'test'. I know it exists, but  i got some error while connecting to it.

db2 => connect to test
SQL1013N  The database alias name or database name "TEST" could not be found.
SQLSTATE=42705

I then listed all databases in the machine and to my surprise i didn't find 'test' db.

db2 => list db directory
SQL1057W  The system database directory is empty.  SQLSTATE=01606

Now i tried creating the same db again and got the following error.

db2 => create db test
SQL1005N  The database alias "TEST" already exists in either the local database
directory or system database directory.

I googled and found a clue which made me to issue 'catalog' command and that solved my problem. After issuing the catalog command i was able to delete the DB. Issue solved!!

db2 => catalog db test
DB20000I  The CATALOG DATABASE command completed successfully.
DB21056W  Directory changes may not be effective until the directory cache is
refreshed.

SQL Server - SQLCMD & OSQL utilities

I had to run a SQL file that was more than 100 MB in size with few lacs of record inserts. I somehow copied the entire file contents to SQL Server Query Analyzer GUI and clicked the 'Execute' button. I got OutOfMemory error. Then i looked for a command line utility to execute those queries. I got OSQL.EXE and it was very much helpful for me. The exe was located at C:\Program Files\Microsoft SQL Server\90\Tools\Binn.

 Command Usage:

osql -Usa -Psa123 -i script.sql

Where,

sa is the user and sa123 is the password and script.sql is the DDL.

 More Links:

  • http://www.databasejournal.com/features/mssql/article.php/3403331
  • http://msdn2.microsoft.com/en-us/library/aa214012(SQL.80).aspx

There is one more utiliy 'SQLCMD' which does the same task. I found out that SQLCMD was introduced to replace osql utility.

JAXB - XMLAccessType, AccessType confusion

Older versions of jaxb-api.jar would generate files with AccessType class whereas new jaxb-api.jar (2.1 version onwards) would generate files with XMLAccessType class reference. If you get any exception related to XMLAccessType (or AccessType) make sure that appropriate jars are used.

classnotfoundexception vs noclassdeffounderror

I got a NoClassDefFoundError and i tried to remember the difference between this error and ClassNotFoundException. Unfortunately I was not able to get the difference and i googled and got the following links:

- http://www.jroller.com/sjivan/entry/difference_between_classnotfoundexception_and_noclassdeffounderror

- http://javafanatics.blogspot.com/2007/03/noclassdeffounderror-vs.html 

  • NoClassDefFoundError is the result of the futile implicit load of the
    class (Class1). Meaning the system classloader is trying to load the
    class Class1 due to inherent call by Class2 (may be an inheritence or
    composition there).
  • ClassNotFoundException comes up when there is an explicit load, using ClassLoader.loadClass, Class.forName etc.

Struts 1.1 vs 1.2 vs 1.3

After a long gap, I tried with Web application using Struts. I had few initial hiccups before getting the index.jsp page to work. One apparent mistake was, I used incompatible softwares. I took sample code from an old ebook which was referring to Struts 1.1. But the jar files that I had was Struts 1.2.

I got the following error when I tried to access index.jsp:

 Attribute name invalid for tag form according to TLD

I googled and found the solution. The '<html:form>' tag was using 2 attributes that are not required any more. I removed those attributes and everything went fine thereafter. This is one change that happened in Struts 1.2.

Hence, it is always good to know the changes that happen from one release to another.

MySQL configuration in Unix

After installing MySQL in AIX box, I struggled to start to daemon. I got lot of errors like the one below:

$ ./bin/safe_mysqld
Starting mysqld daemon with databases from /usr/mysql/data
STOPPING server from pid file /usr/mysql/data/Everest.pid
080211 15:22:50  mysqld ended

 
Even after surfing the net, I was unable to start MySQL daemon. I found a file 'INSTALL-BINARY' in MySQL folder and read the contents.

Here is what is available in the file:

The basic commands that you must execute to install and use a MySQL binary distribution are:
     shell> groupadd mysql
     shell> useradd -g mysql mysql
     shell> cd /usr/local
     shell> gunzip < /PATH/TO/MYSQL-VERSION-OS.tar.gz | tar xvf -
     shell> ln -s FULL-PATH-TO-MYSQL-VERSION-OS mysql
     shell> cd mysql
     shell> chown -R mysql .
     shell> chgrp -R mysql .
     shell> scripts/mysql_install_db --user=mysql
     shell> chown -R root .
     shell> chown -R mysql data
     shell> bin/mysqld_safe --user=mysql &


I simply followed the instructions and i got it working!!

JSR 181: Web Services Metadata

JSR-181, which defines eight annotations to enable easy definition of Java Web Services in a J2EE container, has been approved with a 100% approval.

Included annotations are:

  • javax.jws.WebService
  • javax.jws.WebMethod
  • javax.jws.OneWay
  • javax.jws.WebParam
  • javax.jws.WebResult
  • javax.jws.HandlerChain
  • javax.jws.soap.SOAPBinding
  • javax.jws.soap.SOAPMessageHandlers
