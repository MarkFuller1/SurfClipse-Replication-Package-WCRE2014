Java.io.FileNotFoundException: ... (The system cannot find the path specified)

Symptom

The Software is complaining that it cannot find a file or directory at run time.

Solutions

Scenario 1

Be careful with relative pathnames in Map Publisher. Make SURE that you do not put a back-slash before the root install directory (e.g., "\config\seat\mapsets\WindMil\translator\dumps\Porche Config\PORSCHFL.csv" should instead be "config\seat\mapsets\WindMil\translator\dumps\Porche Config\PORSCHFL.csv"). Placing a backslash before the root install directory will cause the Publisher to look at the root directory of the hard drive instead of the root directory of Partner. Log snippet follows:

Scenario 2

This Window pops up when installing Partner onto a custom folder:

Solution 2

This had to do with permissions on Windows 7, 64-bit machine. The "Partner/" folder did not have a user name "Everyone" under security permissions (Right-click on the folder, go to properties, and click on the "Security" tab). The "Everyone" user group had to be added and given full control permissions. That resolved this particular case.
