Re: Using plugin-in : Seadragon exporter

by mbastian » 04 Apr 2011 02:48

For using the plugin from the Toolkit, you actually only need org-gephi-plugins-seadragon.jar, the others are related to UI.

You can look how the UI is calling the right instructions and use that to do the export programmatically.

I would do
Code: Select all
SeadragonExporter exporter = new SeadragonExporter();
exporter.setPath(new File("folder"));

Workspace currentWorkspace = Lookup.getDefault().lookup(ProjectController.class).getCurrentWorkspace();
exporter.setWorkspace(currentWorkspace);

exporter.setWidth(4000);
exporter.setHeight(4000);
exporter.setTileSize(256);
exporter.getPdfExporter().setMarginBottom(20);
exporter.getPdfExporter().setMarginTop(20);
exporter.getPdfExporter().setMarginLeft(20);
exporter.getPdfExporter().setMarginRight(20);

exporter.execute();


Seadragon exports uses same parameters than PDF, so it uses Preview. You can look at this page how to set these settings.

Re: Using plugin-in : Seadragon exporter

by mbastian » 05 Apr 2011 07:07

Actually, the PDRRenderer library, which is included in the seadragon plugin is missing. That's why you get this error, sorry about that.

I figured out that the UnpackNBM application was not extracting the library from the NBM. I fixed this problem, and you can use the 0.2 version of UnpackNBM. You will get a PDFRenderer.jar library to include in your classpath and it should be working fine.

The sources of the plugin are here btw:
Code: Select all
https://code.launchpad.net/~gephi.team/gephi/seadragon
