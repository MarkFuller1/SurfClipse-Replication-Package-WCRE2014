Matt Burgess added a comment - 07/Aug/13 11:00 AM - edited
I looked into this, it's not a regression from PDI-9900 in the sense that referred to Jobs and the fix for that case did indeed fix the issue for Jobs. However it is a regression from 4.8, the logging string isn't being encoded as CDATA for Transformation Status (although it is for Jobs)

Fix is to encode the logging string as CDATA using the XMLHandler.buildCDATA() method as is done in SlaveServerJobStatus.
