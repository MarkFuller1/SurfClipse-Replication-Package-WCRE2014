If you make a plugin with ReProccer and allow Wrye Bash to read that plugin then you are doing it wrong. All dynamically created plugins made with apps such as ReProccer or SkyProc are loaded after the Bash Patch and never loaded into TES5Edit or included in the Bash Patch.

In response to post #7724728. #7745608, #7771536 are all replies on the same post.

I have realized the issue and found a fix. I looked at how ASIS managed to do this and have duplicated the method. Create a .bat in your ReProccer folder and copy/paste the following...

@ECHO OFF
SET BINDIR=%~dp0
CD /D "%BINDIR%
REM If this line below fails, replace it with the location of your java 7 install.
"C:\Program Files (x86)\Java\jre7\bin\javaw.exe" -Xmx1024M -Xms1024M -jar ReProccer.jar
PAUSE

save the file and run it. It will load up the ReProccer correctly.
