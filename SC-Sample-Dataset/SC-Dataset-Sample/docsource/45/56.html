I can recreate this error.  The way I was able to do this was to first isntall UDS using the FQDN of the box as the Core URL.  Once the install completed I uninstalled UDS and then reinstalled it connecting to the same database.  This time I used localhost for my Core URL.  After the install completed I get the same error as you.  This is because the subject of the certificate used to communicate with the Core in my GUI keystore is the FQDN from my original installation. 

 

I was able to work around this by opening the config.properties found by default at:

 

C:\Program Files (x86)\Research In Motion\Universal Device Service\RIM.BUDS.Gui\webapps\ROOT\WEB-INF\classes

 

and editing the mdm.restServer value.  I changed this from localhost to the FQDN that I originally installed using that matches what is in the certificate.  After restarting the service I was able to login.

 

However you really shouldn't be using localhost for the Core URL.  If you've never logged into the GUI since installing I would suggest reinstalling with a new database and using the FQDN for the Core URL.
