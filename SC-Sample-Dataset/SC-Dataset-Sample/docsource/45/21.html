I resolved the problem!!!!!!
I have my server (with Liferay and Cas) in a machine with Ubuntu as SO.
I follow the guide (Single SignOn - Integrating Liferay With CAS Server) and so the cn for my certificate in cacerts is <hostname>.
My computer has windows as SO.
The DNS server in my lan can't reach to resolve the hostname in ip adress and so i had the problem posted before.
To resolve this i add in the file host (path: C:\WINDOWS\system32\drivers\etc) the pair
<ip address server> <hostname server>.
In this way my machine resolve directly my hostname.
I hope to help you.

sharing my experience in resolving the same issue with liferay 5.2.3 on windows.

Make sure which JRE your liferay is using
-- default JRE on your machine
OR
-- the JRE that comes in liferay bundle at liferay-portal-5.2.3\tomcat-6.0.18\jre1.5.0_17

Then import the certificate to the correct JRE's store - <JRE location of your liferay tomcat >\win\lib\security\cacerts

In case if you like your liferay to use another JRE, below property can be changed from liferay-portal-5.2.3\tomcat-6.0.18\bin\setenv.bat on windows.

set JRE_HOME=%CATALINA_HOME%/jre1.5.0_17/win

happy working with liferay !!!
