Well, I hoped someone else with more knowledge would reply to this, because I really don't think I have enough knowledge to help much...

One question: have you tried using the old plugins on the new Eclipse directly? i.e. just drop all of the plugins in the dropins folder of your installation? I don't expect that to work really, but it is worth a try.

If you develop plugins in the "Eclipse for RCP and RAP Developers" from http://eclipse.org/downloads/ then you will automatically be developing plugins using the compatibility layer. In order to develop plugins for Eclipse 4.x natively you have to install the e4 development tools as described in http://www.vogella.com/articles/EclipseRCP/article.html which is recommended against in said article due to the fact that the APIs are still provisional.

This will be an exceedingly manual process, but I would recommend using http://java.decompiler.free.fr/?q=jdgui to get the source out of the plugins that you want to migrate and then importing them into projects in Eclipse and seeing what doesn't compile. Assuming that Eclipse has been good at maintaining backwards compatibility, there shouldn't be too many changes that you need to make.

I only have very minor Eclipse plugin development experience so I can't really help you on specifics.
