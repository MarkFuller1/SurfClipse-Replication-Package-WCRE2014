prog12

November 13, 2007

prog12 is here. In case you missed class today, the notes I handed out are here. Please review them before lab period.

  • I have added MyBrowser.java. It is a bit clunky, but it works with my solution to MyGoogle.java.
  • I have added Main.java to test your MyGoogle.java.
  • I strongly suggest you start your gather loop like this:

int count = 0;
while (!urlQueue.isEmpty() && count++ < 100) {
String url = urlQueue.poll();
System.out.println(url);

This way you will see each URL as it is processed and your program won't take forever.

  • To create an array of Iterator, the incantation is

Iterator[] pageIDiter = (Iterator[]) new Iterator[n];

Ha ha! WordPress won’t let me tell you the incantation. The first and second “Iterator” on this line should be “Iterator lessthan Integer greaterthan”!

  • Make sure the class PairID that you are putting into your PriorityQueue is declared to implement comparable lessthan PairID greaterthan.

  2. vjm wrote:

    Gather is everything up to the word SEARCHING in the notes, and search is everything after.

    In the gather phase, you start from a list of URLs and follow all links from these pages and follow all links from those and follow all links from those and so on until you have seen every page that is reachable from the starting URLs. As you visit each page, you increment the number of references for each page it links to. For every word on that page, you record that that page has that word.

    In the search phase, given some keywords, like Victor Milenkovic Java, you find all pages which have all three of those words and return an array of their URLs, ordered by decreasing significance, where the significance of a page is the number of links to it from other pages.
