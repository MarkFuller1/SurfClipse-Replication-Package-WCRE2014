This is your original code, with a few more lines marked: ArrayList<Integer> tempCommonWatchedMovies = new ArrayList<Integer>();         for (int i = 2; i < watchedMoviesSet.size(); i++) {             ArrayList<Integer> tempWatchedMovies = watchedMoviesSet.get(i);             for (int movie : tempWatchedMovies) { ***            for (int commonMovie : commonMovies) {                     if (movie == commonMovie) { @@@                    tempCommonWatchedMovies.add(movie);                     }                 }             } ###         commonMovies = tempCommonWatchedMovies;         }  Although your array version might work, I'll explain the problem with your original ArrayList version.  The line that I marked with ### makes 'commonMovies' have the same value as 'tempCommonWatchedMovies'. For i = 2, your loop will go through without exceptions. At that point, 'commonMovies != tempCommonWatchedMovies', and therefore the 'add' on the line with @@@ does not affect the same list. However, for i = 3, 'commonMovies' points to the same list as 'tempCommonWatchedMovies' (i.e., "commonMovies == tempCommonWatchedMovies" is true). Then, when you iterate over 'commonMovies' on line ***, you are iterating over the same list to which you add items on line @@@. Thus, the ConcurrentModificationException occurs. Depending on what is in the original watchedMoviesSet, the error might not occur on i = 3. The error will occur for the first 'i' for which the line @@@ is called.  For i >= 3, your loop where the error occurs is effectively: ***            for (int commonMovie : tempCommonWatchedMovies) {                     if (movie == commonMovie) { @@@                    tempCommonWatchedMovies.add(movie);                     }                 }  or, equivalently, ***            for (int commonMovie : commonMovies) {                     if (movie == commonMovie) { @@@                    commonMovies.add(movie);                     }                 }   When written that way, it may be more obvious as to why the ConcurrentModificationException occurs--you iterate over the list to which you are adding items.  Instead of line ###, you could try this: commonMovies = new ArrayList<Integer>(tempCommonWatchedMovies);  Then you hold the same values in both lists, but the lists themselves are different, and the ConcurrentModificationException will not occur.  You could also try looking at the retainAll method of ArrayList. You'll probably want to make some copies of the list (as in my example code) to avoid corrupting the original lists (if you need the originals to still be intact). But, retainAll should allow you easy ways of retaining only common elements. Underneath, retainAll will still have to do the loops that you are doing, but your code will be shorter, because the loops themselves no longer appear in your own code. You'll just have a few copies and a few calls to retainAll.

doremifasollatido wrote: Again, look at retainAll to see if it simplifies your code.  Note, however, that retainAll will give O(N^2) performance when using two Lists. This won't be a problem for smaller lists (say, under 100 elements each) in a run-once-and-exit educational exercise, but will likely be unacceptable in a production environment. Also, it may or may not give the behavior you want if there are duplicates within a list.
