Iterators in Java (Thread safe Discussion, bidirectional traversal &  Fail-safe and Fail-fast)

Vaibhav Kashyap
An Iterator is used to iterate over a Collection. It was brought as a improvement to Enumeration. It differs from Enumeration in two ways:
a) Methods names are smaller
b) Method for removing elements from Collection was added.

Lets understand its functioning with the help of an example :

/**
 *
 * @author vaibhav
 */
import java.util.ArrayList;
import java.util.Iterator;
import java.util.List;
 
public class IteratorDemo {
 
    public static void main(String[] args) {
 
        List<Integer> myList = new ArrayList<Integer>();
        myList.add(0);
        myList.add(1);
        myList.add(2);
 
        Iterator<Integer> itr = myList.iterator();
        while(itr.hasNext())
        {
           Integer i = itr.next(); /* next() returned object of Integer type since
          we used Generic Iterator*/
          // int i = (Integer)itr.next();  if generic Iterator is not used
           System.out.println(i);
        }
    }
}

Elements are added into the List(Interface) with the help add method of List & for traversing we have used Iterator.
"Iterator<Integer>" : Iterator are also available in generic form. For better programming practice try to use generic Iterator instead of normal one. Generic Iterator enforces acceptance of data of particular data type for e.g Integer in my case.
Now we have used two important methods :

a) hasNext() : returns a boolean value , true or false. True if more elements are available to be traversed and False if nothing is available for traversal.

b) next() : returns object of Object class. You need to type caste it to use. if simple Iterator is used e.g
Iterator itr = myList.iterator();


Iterators are Thread-Safe ??
No , For ArrayList, LinkedList, HashMap you can get a ConcurrentModificationException if the list is modified when you are iterating over it. Although it is not guaranteed but i have personally faced this problem while working with threads.
Everywhere i have seen people have suggested to use Iterator over for-each loop. No doubt it will work. But are you sure ??
For e.g.

package IteratorDemo;

/**
 *
 * @author vaibhav
 */
import java.util.ArrayList;
import java.util.Iterator;
import java.util.concurrent.CopyOnWriteArrayList;

public class IteratorTest {

ArrayList<Integer> arr=new ArrayList<Integer>();
public static void main(String... q)
{
    new IteratorTest().show();
}
public void show(){

    arr.add(2);
    arr.add(5);
    arr.add(9);
    Iterator i=arr.iterator();
    while(i.hasNext())
    {
        arr.add(10);
        System.out.println(i.next());
    }
}
}


You'll get :

run:
Exception in thread "main" java.util.ConcurrentModificationException
at java.util.ArrayList$Itr.checkForComodification(ArrayList.java:819)
at java.util.ArrayList$Itr.next(ArrayList.java:791)
at IteratorDemo.IteratorTest.show(IteratorTest.java:31)
at IteratorDemo.IteratorTest.main(IteratorTest.java:20)
Java Result: 1
BUILD SUCCESSFUL (total time: 1 second)


Now what ???

Java Concurrency API provides two means to deal with such problems:

a) java.util.concurrent.CopyOnWriteArrayList
b) java.util.concurrent.ConcurrentHashmap

Lets re-write the above code , just replace ArrayList with CopyOnWriteArrayList.

CopyOnWriteArrayList<Integer> arr=new CopyOnWriteArrayList<Integer>();

you'll get :

run:
2
5
9
BUILD SUCCESSFUL (total time: 1 second)

magic behind the replacement is cloning. CopyOnWriteArrayList creates internally a clone of the list and then performs the operation over the clone which avoids ConcurrentModificationException.

Bi-Directional Traversal In Iterator :
Normal Iterator provides only forward traversal and provides remove() for removal of elements but what to do if bidirectional traversal is required ??
Use java.util.ListIterator for such requirements.

For e.g.
public static void main(String args[]) {
        List<String> list = new ArrayList<String>();
        list.add("vaibhav");
        list.add("anvesh");
        list.add("rohit");
        System.out.print("Originlist contents of list:\n ");
        for(String str:list){
            System.out.print(str);
        }
       
        ListIterator<String> litr = list.listIterator();
        while (litr.hasNext()) {
          String element = litr.next();
          litr.set(element + "+");
        }
       
        System.out.print("\nModified list backwards: ");       
        for(String str:list){
            System.out.print(str);
        }
    }

Fail-Safe & Fail-Fast :

Fail-fast is identified internally using a modification count for the collection and if the iterator realizes that the count is changed, it will thrown a ConcurrentModificationException , which is already discussed before.

Fail-safe are similar to fast-fail but they do not throw an Exception when Collection is modified since they work on a clone of the collection. CopyOnWriteArrayList and ConcurrentHashMap are fail-safe iterators which work on a copy of a collection for modification.
