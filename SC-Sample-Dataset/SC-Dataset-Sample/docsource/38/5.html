Jave client crashes with URE: java.lang.reflect.UndeclaredThrowableException in Rational Synergy

Technote (troubleshooting)


Problem(Abstract)

Attempts to start a Jave/Developer client crashes with "Unhandled runtime exception(URE): java.lang.reflect.UndeclaredThrowableException" in IBM Rational Synergy.

Symptom

The Rational Synergy client will not start in web mode on a particular client machine. The same user can start a session on the same database using a different client machine.

Cause

This can happen if a patch has been installed incorrectly. Specifically if the ccmjava.jar in the client is not at the correct patch level for the client installation, the server installation, and the database.

Environment

Rational Synergy web mode

Diagnosing the problem

Check the synergy.log in %CCM_HOME%\log on the Server. On the client the ccmclient.log will note the runtime exception as well.


ccmclient.log

(date) An unhandled runtime exception has occurred.  See synergy.log for more


The patch level noted just prior to this error will have the incorrect patch level in comparison to the server. Notice how the patch goes back to an older version after noting 7.1.0.4 in the example:

(date) 7.1.0.4 IBM Rational Synergy 7.1.0.4 Fix Pack
(date) 7.1.0.4.002 IBM Rational Synergy 7.1.0.4 iFix002
(date) 7.1.0.1 This is Patch 7.1.0.1 for IBM Rational Synergy Release 7.1
(date) 7.1.0.1 This is Patch 7.1.0.1 for IBM Rational Synergy Release 7.1

Resolving the problem

Re-install the client and reapply all patches up to the latest patch level on the server and databases.

You cannot assume that only the ccmjava.jar file is incorrect, so it is important to verify all of the files in the installation have been patched correctly.
