Content Does Not Start After Reverting Back To The Changes Made In dssetup

 

Note:  Xerox Internal Use Only.

 

Note:  This applies to DocuShare 6.5.x or DocuShare 6.6.x.

 

Issue

 

Content does not start after reverting back to the changes made in <dshome>\bin\dssetup.

 

The <dshome>\logs\Monitor.log file displayed an error similar to the following:

 

Exception exception java.rmi.ServerException: RemoteException occurred in server
 thread; nested exception is:
        java.rmi.RemoteException: IndexServerImpl: Exception instantiating index
er class: com.xerox.docushare.DSException: IdolIndexer.init: Caught exception: c
om.xerox.docushare.DSException: content server is not running.: content server
 is not running.

 

OR

 

18 Aug 2011 01:22:46 [main] ERROR - MonitorServerImpl.getSystemStatus: content
 has failed

 

Possible Cause / Reason

 

This happens when changing IDOL ports using the dssetup  command and then trying to change the settings again.  The dssetup command will successfully change the IDOL ports on the first attempt.  However, if you try to change them again the dssetup command will fail to change the IDOL port on the second attempt.

 

Solution for DocuShare 6.6.1

 

This issue has been fixed in DocuShare 6.6.1 Patch 1. 

 

Workaround for DocuShare 6.5.x

 

Change the IDOL ports manually by editing the following files.

·         <dshome>\IDOLServer\IDOL\AutonomyIDOLServer.cfg

·         <dshome>\IDOLServer\FileSystemFetch\AutononyFileSystemFetch.cfg

·         <dshome>\IDOLServer\DiSH\AutonomyDiSH.cfg

 

Note:  This will be fix in the next patch release.
