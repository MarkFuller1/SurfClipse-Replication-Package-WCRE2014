  4. Re: Overflow exception

    > My classes implement toString(). Why the classes couldn't implement this
    > method?

    There are certain situations when trying to access a persistent field cause=
    s an
    exception (for example, the object is deleted). JDO mandates that the exce=
    ption
    we throw has to include a reference to the object that caused the exception=
    ..
    When the stack trace prints, the JDO exception tries to print this failed
    object. If calling toString() on the object tries to access a persistent f=
    ield
    again, you get a new exception. And so forth.

    We've done some work in our latest betas to work around the fact that the
    standard javax.jdo exception types can result in errors when you try to pri=
    nt
    them. But for now, please try the workaround Steve suggested.
