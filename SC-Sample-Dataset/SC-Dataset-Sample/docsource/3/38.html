java.lang.NumberFormatException

java.lang.NumberFormatException is very common exception during development while developer is trying to convert String to Integer.

In this Tutorial We will show you the reason of Number Format Exception, finding the line of code in JAVA program that cause this exception and solution to fix this exception.

java.lang.NumberFormatException Example:

         
package com.Jwalant;

public class NpeExample {

	public static void main(String[] args) {
		
		String[] testArray = {"7", "4", "Hello", "9"};
		
		for(int i=0; i< testArray.length; i++){
			System.out.println(Integer.parseInt(testArray[i]));
		}
	}
}

Output on console:

7

4

Exception in thread "main" java.lang.NumberFormatException: For input string: "Hello"

at java.lang.NumberFormatException.forInputString(Unknown Source)

at java.lang.Integer.parseInt(Unknown Source)

at java.lang.Integer.parseInt(Unknown Source)

at com.Jwalant.NpeExample.main(NpeExample.java:10)

 

As you can see above, we tried to convert number from String.

Since first 2 Strings are "7" and "4", it can be converted to the Integer by calling static parseInt() method of Integer class. The 3rd element is "Hello" and can not be converted to Integer and it throws the Number Format Exception.

Find the line in the Code that throws java.lang.NumberFormatExceptionfrom exception stack trace:

In the example above, we intentionally developed line of codes that throws Number Format Exception. However in real time, you do not know the line of code until it throws Number Format Exception and breaks the software.

Hence, it is important to identify the line of code that throws java.lang.NumberFormatException and fix it. So the question is, how do we know which line throws the exception? Well, the answer is from Stack Trace. You can find the log or go to output console and find the stacktrace.

In the above example, we also provided stack trace as output. It looks like this:

            at com.Jwalant.NpeExample.main(NpeExample.java:10)

As you can see above, it says that Exception is thrown at NpeExample.java:10. It means, in the class NpeExample.java line number 10 throws the exception. 
If you look at line number 14, it is 
            System.out.println(Integer.parseInt(testArray[i]));
Now, it is easier for us to apply fix on this line.

Fix the line in the Code that throws java.lang.NumberFormatException:

 
Once we find out the line in the code that throws java.lang.NumberFormatException, we want it to fix.

For now, we will show you a solution to fix java.lang.NumberFormatException.

Solution 2: Use try catch block to catch exception. For example, you can write following line of code:

         try {
               System.out.println(Integer.parseInt(testArray[i]));
           }catch (Exception e) {
                e.printStackTrace();
                // TODO: handle exception
           }
This statements will not break the software or interrupt the code execution. You can still print the Exception stack trace to the output console or log using printStackTrace() function of Exception class.
