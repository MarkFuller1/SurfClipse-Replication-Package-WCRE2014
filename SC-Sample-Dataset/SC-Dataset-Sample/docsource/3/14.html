Anand Mehta added a comment - 11-May-2011 14:33
Thanks, Savitha. This helped in resolving the issue with a little change to the SQL statement as listed below:

Steps 1-3 are needed if the xml field is of type Long (it was in our case)

1. alter table announcement_message modify xml clob;
2. select 'alter index '||index_name||' rebuild online;' from user_indexes where status = 'INVALID' or status = 'UNUSABLE';
3. execute all statements from step# 2
4. update ANNOUNCEMENT_MESSAGE set MESSAGE_ORDER='1', XML=REPLACE(XML, ' subject=', ' message_order="1" subject=') WHERE MESSAGE_ORDER is null;
