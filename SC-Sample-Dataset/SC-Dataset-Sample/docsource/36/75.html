You didn't say what type of project you were working on.

If you're doing something like the JDBC tutorial:

http://download.oracle.com/javase/tutorial/jdbc/basics/index.html

You'll need to add the jar file or your newly created library to the project.
What you've done is created a library so that all projects in NetBeans can
easily add the MySQL driver.

To add a library to your project:

1. Right-mouse click on the Libraries folder in your project
2. Select Add Library
3. Scroll down and find the Library you need
4. Select it, and click on the Add button

If you are working with a JAR file instead, it's slightly different.

In step 2 above you'll select Jar/File Folder.

In step 3 above, you'll have to browse to the location. You'll be given a
choice for absolute or relative paths. I usually use relative paths, since I
put foreign jars in a special folder within the project (SCM folk, shoot me
later).

In step 4 above, you'll click on Open.

You can also add libraries and jar files via the project properties selection.
This gives you a bit more control (phase of project, order of jar files or
libraries).

1. Right-mouse click on the project
2. Scroll down to properties
3. Click on Libraries in the Categories panel
4. Add as appropriate (usually in the Compile tab)

If you're writing a J2EE application that runs on Tomcat or Glassfish, it's
probably better to use JNDI and a container-managed connection pool.

Explaining how to do that is container-specific.

See the following:

http://tomcat.apache.org/tomcat-6.0-doc/jndi-resources-howto.html
http://download.oracle.com/docs/cd/E19798-01/821-1751/ablih/index.html

The code to access JNDI resources is a little different (see the example in
the Tomcat documentation), but ultimately simpler.

Some other good notes about managing database connections can be
found here:

http://blog.christopherschultz.net/index.php/2009/03/16/properly-handling-pooled-jdbc-connections/

. . . . just my two cents
