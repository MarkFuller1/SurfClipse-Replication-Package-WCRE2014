Creating a DataSource with SQLLite

up vote 0 down vote favorite
1

It's clear from the documentation how to create a database connection, ie:

Class.forName("org.sqlite.JDBC");
SQLiteConfig config = new SQLiteConfig();
config.setSharedCache(true);
Connection c = DriverManager.getConnection("jdbc:sqlite:/data/mydb.db", config.toProperties());

How do you make a "DataSource" to pass around to places that need such a thing? I know you can do the following (see below), but im not sure about what the performance of this is like:

Class.forName("org.sqlite.JDBC");
SQLiteConfig config = new SQLiteConfig();
config.setSharedCache(true);
SQLiteDataSource ds = new SQLiteDataSource(config);
