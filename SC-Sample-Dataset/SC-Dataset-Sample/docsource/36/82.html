It looks like this error is caused by the invoking classes being loaded in a way that means their classloader are not connected to the ClassPath class loader. A work around might be to explicitly fetch the class loader for something on the path, and use that in the forName call.

Charles Oliver Nutter added a comment - 12/Feb/09 10:31 PM

The problem here is that after 1.1.3, JRuby's classes are being loaded into the bootclassloader, and since Class.forName uses the classloader of the caller, classes on $CLASSPATH are not visible. The workarounds are simple, use import or include class, or reference the class directly, as org.postgresl.Driver. I've added a FAQ to the wiki here: http://wiki.jruby.org/wiki/FAQs#Why_do_I_get_ClassNotFoundException_when_I_call_java.lang.Class.forName_from_Ruby.3F
