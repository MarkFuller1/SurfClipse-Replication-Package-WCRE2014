    Being able to change the security policy from within an application kind of defeats the purpose of security

    Create a security policy file, ie server.policy and put something similar to the following in it:
    Code:
    grant
{
        permission java.net.SocketPermission "*:1024-65535", "connect,accept";
        permission java.lang.RuntimePermission "setSecurityManager";
        permission java.lang.RuntimePermission "createSecurityManager";
};
    When you run your application, specify the new policy with:
    java -Djava.security.policy=server.policy <YourClass>
