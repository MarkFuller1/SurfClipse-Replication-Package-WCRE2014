  RMI (Remote Method Invocation) 

  The series will be introduced with the other two different RPC programming interface, RMI (Remote Method Invocation) was quite old, which is Java-IDL were introduced before joining the J2SE.    RMI and CORBA same development process (from the time unable to determine whether RMI is customized according to CORBA Specification), and its relatively cumbersome development process, but because RMI is the basis of EJB, therefore, it is in the development of Java very important position. 

  Following is the creation of Remote Method Invocation five steps: 

  1. Expansion of the definition of a Remote Interface interface for each one of the ways must declare that it will produce a RemoteException anomaly; 

  2. Realization of the definition of an interface; 

  3. Rmic generated using remote achieve the required stubs and frameworks; 

  4. Create a client and server RMI call; 

  5. Rmiregistry start and run their own procedures and customer service procedures.
