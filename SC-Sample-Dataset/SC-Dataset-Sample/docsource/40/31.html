@episod Taylor Singletary

Hi everyone.

On your app settings screen, the presence of a "default" or placeholder callback URL determines whether your application can do callback-based OAuth or not. If no placeholder is present, you can only do out-of-band OAuth (or xAuth if you have access). If the placeholder is present, you're allowed to do callback-based auth. (We're working to simplify this distinction).

Either way, on the request token step of OAuth, you should be sending a dynamic oauth_callback parameter regardless of what you've set on your application settings page.

@episod Taylor Singletary

The callback field on dev.twitter.com is more of a placeholder than anything else. Don't think too hard about it and just put something that's not a twitter.com but still a valid HTTP-based URL like "http://google.com" -- by putting a value there, it allows you to specify a dynamic callback URL on the oauth/request_token step. The value you put in on that step is much more important and relates directly to whatever you've programmed to handle the callback.
