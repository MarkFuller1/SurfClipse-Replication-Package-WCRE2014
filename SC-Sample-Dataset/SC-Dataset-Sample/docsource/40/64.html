    java.net.ProtocolException: Server redirected too many times

    Hi,

    I am getting a java.net.ProtocolException: Server redirected too many
    times (20) at the method
    sun.net.http://www.protocol.http.HttpURLConn...getInputStream.

    I added the proxy server settings to the JVM parameters in WebSphere,
    which solved this problem, however, our architect doesn't want this
    proxy setting in WebSphere and says that it is a network issue.

    Does anyone know how to resolve this without using the proxy
    parameters?

    Any help or ideas would be greatly appreciated as we have production
    go-live very soon.

    Thanks,

    I have solved this already. I simply put the proxy parameters inside
    the code. ...made sense.

    systemProperties = System.getProperties();
    systemProperties.setProperty("http.proxyHost", proxyHostID);
    systemProperties.setProperty("http.proxyPort", proxyPortID);
    systemProperties.setProperty("http.nonProxyHosts", nonProxyHostsID);
