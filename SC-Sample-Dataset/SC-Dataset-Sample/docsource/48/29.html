Jeethendriya,
I would suggest that you create a new workspace, it looks like part of your workspace files got corrupted. You can try and use the -clean command line argument when starting Eclipse to see if that helps.

Also make SURE you are not using gcj to run MyEclipse, it is not tested or supported at this time, and I know that RedHat likes to ship that as the default Java environment. Also if you are using the natively compiled Eclipse, that is not supported at this time either.

1) Your compiler failed messages are because "javac" is not in your system path, so MyEclipse cannot find it to validate your JSP pages.


2) A slew of those exceptions are from another plugin that is not shipped with MyEclipse.


3) The last exception, java.lang.NoClassDefFoundError: org/eclipse/core/resources/IProject , is serious. That is a core Eclipse class, if you are getting not found exceptions with that, then something is definately wrong either with the workspace or the install. We have quite a few users (About 15%) running on Linux and also test on Red Hat linux regularly and haven't seen consistent crashing or corruption when using the correct versions of software.


** Be sure you are not using the natively compiled Eclipse that RedHat includes in it's distro. Also make sure you aren't using GCJ as I mentioned above. Redownload Eclipse SDK 3.1.1 if you have to from eclipse.org and also MyEclipse 4.0.3 from our site; additionally we encourage our users to use Sun's JDK for linux as well to get the most stable setup.
