java.lang.IllegalArgumentException: object is not an instance of declaring class


As I was typing up this email, it occurred to me that the name of the component class, without the fully qualified package, gets cached in some way? To test this theory, I renamed org.mule.example.hello.BOS_1_5_0.VersionComponent and org.mule.example.hello.BOS_1_6_0.VersionComponent to org.mule.example.hello.BOS_1_5_0.VersionComponent1_5_0 and org.mule.example.hello.BOS_1_5_0.VersionComponent1_6_0, respectively. Magically, this seems to have fixed my issue.

Thoughts?

-----

In the following config, when sending in files as input, it seems the first message received, gets routed; but, the second message received results in the IllegalArgumentException. I've tried changing the order of messages and see the same behavior (i.e. both 1.5.0.xml and 1.6.0.xml may result in the same IAE depending on which comes first).
