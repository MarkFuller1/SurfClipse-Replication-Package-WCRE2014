      Not sure exactly why you're getting that error, but I get it when I declare a transaction & don't tell the transaction proxy the right interface that it's supposed to pretend to be. Like this:

      Code:
         <bean id="personMasterUtilDAO" class="org.springframework.transaction.interceptor.TransactionProxyFactoryBean" parent="txDaoProxyTemplate">
       <property name="proxyInterfaces"><list><value>org.dcjs.sjs.jdbc.PersonMasterUtilDAO</value> </list></property>
       <property name="target"><ref bean="personMasterUtilDAOTxTarget"/></property>
       <property name="transactionAttributes">
           <props>
               <prop key="consolidate*">PROPAGATION_REQUIRED,-Exception</prop>
               <prop key="insert*">PROPAGATION_REQUIRED,-Exception</prop>
               <prop key="update*">PROPAGATION_REQUIRED,-Exception</prop>
               <prop key="delete*">PROPAGATION_REQUIRED,-Exception</prop>
           </props>
       </property>
   </bean>
      If the interface that it should imitate is actually, say, PersonMasterUtilDAO2 then I will get error like yours,
